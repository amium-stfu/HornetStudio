# IMPLEMENTATION HANDOFF

## Goal
Add the first configuration step for BrokerWidget publishing: a `PublishItems` property that opens a TreeView selection for local HornetStudio items and stores the selected paths for later broker publishing.

## Scope
Included:
- Add a persisted BrokerWidget property for selected local publish item paths.
- Expose the property in the BrokerWidget property editor as `PublishItems`.
- Open a TreeView-style selection dialog/editor for local HornetStudio items.
- Store the selected paths in the widget model and layout persistence.
- Keep behavior limited to configuration only.

Not included:
- No MQTT or ItemBroker publishing yet.
- No handling of external writes to published items.
- No writable/read-only per-item settings.
- No changes to existing `AttachToUI` behavior.
- No BrokerWidget UI redesign.

## Tasks
1. Add a new `FolderItemModel` property, preferably `BrokerPublishedItemPaths`, with an empty default value.
2. Persist and restore `BrokerPublishedItemPaths` in layout import/export paths.
3. Add a BrokerWidget property editor entry labeled `PublishItems`.
4. Implement the `PublishItems` editor action so it opens a TreeView selection of local HornetStudio items.
5. Populate the TreeView from existing local item/registry sources used by the editor.
6. Save selected item paths as a newline-separated, normalized path list.
7. Ensure existing layouts without the property load with no published items selected.
8. Add or update focused tests for model default, persistence roundtrip, and property/editor wiring where existing tests allow it.
9. Update BrokerWidget documentation/YAML sample only to mention the configuration property, without claiming publishing behavior exists yet.

## Technical Constraints
- Chat and planning are German; code, comments, XML docs, UI text, file names, and identifiers are English.
- UI label should be `PublishItems`.
- Internal property should be explicit to BrokerWidget, e.g. `BrokerPublishedItemPaths`.
- Match existing `AttachToUI` selection patterns where practical.
- Do not add new dependencies.
- Do not implement MQTT publishing in this step.
- Public classes or public members changed during implementation need XML documentation.
- Keep changes narrowly scoped to BrokerWidget configuration and persistence.

## Notes
- `PublishItems` should mirror the user flow of `AttachToUI`, but it selects local HornetStudio items rather than remote broker items.
- The selected paths are preparation for a later `HostItemBrokerPublisher`.
- Empty selection must mean that HornetStudio publishes nothing.
- Avoid implying that selected items are already published until the later broker publishing step is implemented.

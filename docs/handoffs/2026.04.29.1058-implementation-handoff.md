# IMPLEMENTATION HANDOFF

## Goal
Create a simple, readable demo project for `Amium.ItemBroker` that can serve as a human-friendly template. The demo client must publish two demo items at 10 Hz.

## Scope
Included:
- Add a new console demo project under `src/Amium.ItemBroker.DemoClient`.
- Publish two demo items through `Amium.ItemBroker.Client`.
- Use a 10 Hz update loop with cancellation support.
- Keep the demo code intentionally clear and template-like.
- Add the project to `HornetStudio.sln`.
- Add or update concise documentation showing how to run the demo.

Not included:
- No external transport implementation.
- No MQTT dependency unless already required by the chosen in-process verification path.
- No changes to broker core behavior.
- No UI integration.
- No release/versioning changes.

## Tasks
1. Create `src/Amium.ItemBroker.DemoClient/Amium.ItemBroker.DemoClient.csproj` as a `net9.0` console project with nullable reference types and implicit usings enabled.
2. Reference `..\Amium.ItemBroker.Client\Amium.ItemBroker.Client.csproj`.
3. Implement a small, readable console program that:
   - Creates an `InMemoryItemBroker`.
   - Creates an `ItemBrokerClientSession` with a stable client id such as `demo-client`.
   - Creates two `Amium.Item.Item` instances with explicit paths, for example `Runtime.Demo.Temperature` and `Runtime.Demo.Pressure`.
   - Publishes initial item snapshots once, including useful parameters such as `Unit`, `Kind`, and `Format`.
   - Updates and publishes only values at 10 Hz.
   - Uses `PeriodicTimer` with `TimeSpan.FromMilliseconds(100)`.
   - Handles Ctrl+C through `CancellationTokenSource`.
   - Writes concise console output that shows current values and how to stop the demo.
4. Keep demo responsibilities separated into small methods/classes only where it improves readability, for example:
   - `DemoItemDefinition` or equivalent simple model.
   - `DemoItemPublisher` or equivalent loop owner.
   Avoid unnecessary abstractions.
5. Add the new project to `HornetStudio.sln`.
6. Update `docs/amium-itembroker-usage.md` with a short `Demo Client` section and the run command:
   `dotnet run --project src/Amium.ItemBroker.DemoClient/Amium.ItemBroker.DemoClient.csproj`
7. If `CHANGELOG.md` is used for relevant non-release changes, add a concise unreleased entry for the demo project.
8. Build at least the demo project. Prefer also building the solution if practical.

## Technical Constraints
- Chat must be German, but code, comments, XML documentation, UI/console text, file names, and identifiers must be English.
- Use existing project patterns: SDK-style `.csproj`, `net9.0`, nullable enabled, implicit usings enabled.
- Do not add new runtime dependencies.
- Use `Amium.ItemBroker.Client.ItemBrokerClientSession`.
- Use `InMemoryItemBroker` for a self-contained in-process demo.
- Use async APIs and avoid `.Result` / `.Wait()`.
- Public classes and public methods must have XML documentation if introduced.
- Keep the implementation directly related to the demo only.

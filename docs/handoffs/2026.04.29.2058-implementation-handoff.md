# IMPLEMENTATION HANDOFF

## Goal
Make BrokerWidget use an automatically generated, persisted, readonly local MQTT client id so users cannot accidentally set it to a remote client id.

## Scope
Included:
- Generate a unique local MQTT client id for newly created BrokerWidget instances.
- Keep existing persisted `BrokerClientId` values unchanged when loading layouts.
- Show `BrokerClientId` as readonly in the property dialog with the visible label `LocalMqttClientId`.
- Keep the persisted property name `BrokerClientId`.
- Update BrokerWidget documentation/help to explain that the id is generated locally.

Not included:
- No MQTT topic contract changes.
- No migration that overwrites existing saved client ids.
- No changes to remote client discovery or attach path storage.

## Tasks
1. Add a small helper for generating BrokerWidget local client ids, for example `hornet-studio-{shortGuid}`.
2. Use the helper when creating a new BrokerWidget in the editor.
3. Use the helper for default BrokerWidget model initialization only when no persisted value exists.
4. Change the BrokerWidget property binding for `BrokerClientId` to readonly while keeping label `LocalMqttClientId`.
5. Keep `BrokerClientId` included in save/load/clone/copy flows.
6. Update BrokerWidget docs/help to describe the generated readonly local MQTT client id.
7. Build `src/HornetStudio.Editor/HornetStudio.Editor.csproj`.

## Technical Constraints
- Chat responses must be in German.
- Code, comments, XML documentation, UI text, file names, and technical identifiers must be in English.
- User-visible error and validation messages must be in English.
- Do not introduce new dependencies.
- Keep persisted property name `BrokerClientId`.
- Do not overwrite existing layout values.

## Notes
- Prefer a stable readable prefix plus a short GUID, such as `hornet-studio-{Guid.NewGuid():N[..8]}`.
- The id must be generated once and then persisted.
- Readonly display is preferred over hiding the field because it helps debugging.

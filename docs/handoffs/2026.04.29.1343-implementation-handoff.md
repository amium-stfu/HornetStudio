# IMPLEMENTATION HANDOFF

## Goal
Restructure the ItemBroker client-side MQTT responsibilities by introducing a reusable MQTT client package and a new dedicated MQTT demo client.

## Scope
Included:
- Add a new `Amium.ItemBroker.Client.Mqtt` project for MQTT-based client functionality.
- Keep `Amium.ItemBroker.Client` transport-neutral.
- Keep `Amium.ItemBroker.Mqtt` focused on broker-side MQTT adapter/server functionality.
- Add a completely new MQTT demo client project that consumes `Amium.ItemBroker.Client.Mqtt`.
- Move reusable MQTT client connection/publishing logic out of the current demo implementation.

Not included:
- Do not implement the planned pipe transport yet.
- Do not merge broker-side MQTT adapter code into the client package.
- Do not introduce unrelated refactorings or new dependencies beyond the required MQTT client dependency.

## Tasks
1. Create `src/Amium.ItemBroker.Client.Mqtt/Amium.ItemBroker.Client.Mqtt.csproj`.
2. Reference `Amium.ItemBroker.Client` from `Amium.ItemBroker.Client.Mqtt`.
3. Add the MQTT client dependency required for outgoing MQTT connections.
4. Add MQTT client options, for example host, port, client id, base topic, protocol version, reconnect behavior.
5. Add a reusable MQTT item broker client session or publisher that exposes the existing client-facing publish/write/subscribe API where practical.
6. Reuse or share topic mapping with the existing MQTT mapping behavior where possible without coupling client-side code to broker-side server responsibilities.
7. Create a new demo project, for example `src/Amium.ItemBroker.MqttDemoClient/Amium.ItemBroker.MqttDemoClient.csproj`.
8. Make the new demo project reference `Amium.ItemBroker.Client.Mqtt`.
9. Implement the new demo client so it creates the MQTT client package type and publishes demo items through that reusable API.
10. Keep the existing `Amium.ItemBroker.DemoClient` unchanged unless it must be removed from build wiring or renamed by explicit follow-up request.
11. Add the new projects to the solution.
12. Update relevant documentation or changelog only if project usage/build behavior changes.
13. Build the affected projects and the solution where practical.

## Technical Constraints
- Chat responses must be German.
- Code, comments, XML documentation, UI text, file names, and technical identifiers must be English.
- Public classes and public methods must have XML documentation.
- Keep `Amium.ItemBroker.Client` free of MQTT, pipe, or other transport-specific dependencies.
- `Amium.ItemBroker.Client.Mqtt` may depend on MQTTnet for MQTT client connectivity.
- `Amium.ItemBroker.Mqtt` remains the broker-side MQTT adapter/server package.
- Prefer existing ItemBroker message, path, policy, and topic-mapping patterns.
- Do not add pipe transport implementation in this change.
- Avoid unnecessary abstractions and broad restructurings.

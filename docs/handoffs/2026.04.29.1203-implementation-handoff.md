# IMPLEMENTATION HANDOFF

## Goal
Refactor the ItemBroker demo client so demo items are instantiated directly as `Item Pressure` and `Item Temperature`, and value publishing becomes less repetitive for later usage.

## Scope
Included:
1. Update `src/Amium.ItemBroker.DemoClient/Program.cs`.
2. Remove the `DemoItemDefinition` usage from the demo.
3. Instantiate `Item Temperature` and `Item Pressure` directly in the demo.
4. Keep the demo flow: publish full snapshots once, then publish only `Value` updates at 10 Hz.
5. Fix the current root-path issue caused by paths without a parent segment.
6. Introduce a small demo-local ergonomic publishing helper or item handle if needed.

Not included:
- Do not change the core `Amium.Item.Item` model to know about MQTT or broker sessions.
- Do not introduce new dependencies.
- Do not refactor unrelated MQTT adapter or broker internals.
- Do not change release/versioning files.

## Tasks
1. Replace `_items` / `DemoItemDefinition` with directly instantiated `Item` fields or local variables named `Temperature` and `Pressure`.
2. Assign stable full paths such as `Demo.Temperature` and `Demo.Pressure`.
3. Set item metadata directly on the items: `Unit`, `Kind`, and `Format`.
4. Publish each full item snapshot once before the update loop.
5. During the update loop, update each item value and publish only its `Value`.
6. Reduce repeated `PublishValueAsync(path: ..., value: ..., cancellationToken: ...)` call-site noise, preferably through a small demo-local wrapper/handle that owns the item path and publisher reference.
7. Verify that `PublishValueAsync()` still maps only to the MQTT `.../value` topic and does not publish parameters.
8. Build the demo project or solution after the change.

## Technical Constraints
- Chat responses must be in German.
- Code, comments, XML documentation, UI text, file names, and technical identifiers must remain in English.
- Use existing project patterns and MQTTnet usage.
- Keep changes scoped to the demo unless a tiny reusable helper is clearly justified.
- Public classes or public methods changed or added must have XML documentation.
- Demo MQTT messages may remain non-retained unless the request explicitly changes that behavior.

## Relevant Files
- `src/Amium.ItemBroker.DemoClient/Program.cs`
- `src/Amium.ItemBroker.DemoClient/Amium.ItemBroker.DemoClient.csproj`
- `src/Amium.ItemBroker.Mqtt/MqttTopicMapping.cs`
- `src/Amium.ItemBroker.Mqtt/MqttItemBrokerAdapter.cs`

## Notes
- A snapshot means the full item state: path, value, and parameters.
- `PublishValueAsync()` should publish only the `Value` parameter/topic.
- Avoid putting transport/session state directly into `Amium.Item.Item`; prefer a bound demo item handle if ergonomic instance-style calls are desired.
- The current `DemoItemDefinition.CreateItem()` path handling fails for paths without a dot, so use full paths or remove that code entirely.

# IMPLEMENTATION HANDOFF

## Goal
Remove the redundant `items` segment from MQTT client-owned item topics so demo/client publications use topics like `hornet/clients/demo-client/Demo/Temperature/value` instead of `hornet/clients/demo-client/items/Demo/Temperature/value`.

## Scope
Included:
- Update MQTT topic generation and parsing for client-owned item topics.
- Update client subscription filter for remote item data.
- Update tests that assert MQTT topic strings and inbound topic mapping.

Not included:
- Changing broker health topics such as `hornet/broker/heartbeat`.
- Adding compatibility support for the old `.../items/...` topic format unless explicitly requested.
- Changing demo item names, values, publish frequency, or MQTT connection settings.

## Tasks
1. Update `Amium.ItemBroker.Mqtt.MqttItemTopicMapper` so client-owned topics are built as `<base>/clients/<clientId>/<item-path>/<value|params/...>`.
2. Update `TryMapTopic` in the same mapper to parse the new structure and reject unrelated malformed topics.
3. Update `Amium.ItemBroker.Mqtt.Client.MqttItemTopicMapper.ItemSubscriptionTopic` from `<base>/clients/+/items/#` to `<base>/clients/+/#`.
4. Update all affected assertions and sample incoming publish topics in `src/Amium.ItemBroker.Tests/Program.cs`.
5. Run the relevant ItemBroker tests and then a project/solution build if practical.

## Technical Constraints
- Chat response must be German; code, comments, XML documentation, UI text, file names, and identifiers stay English.
- Keep changes limited to the MQTT topic mapping behavior and directly affected tests.
- Do not introduce new dependencies.
- Preserve existing MQTT escaping behavior with `Uri.EscapeDataString` / `Uri.UnescapeDataString`.
- Preserve broker health topic behavior.
- Public methods/classes touched should retain or update XML documentation as needed.

## Relevant Files
- `src/Amium.ItemBroker.Mqtt/MqttTopicMapping.cs`
- `src/Amium.ItemBroker.Mqtt.Client/MqttItemTopicMapper.cs`
- `src/Amium.ItemBroker.Tests/Program.cs`
- `src/Amium.ItemBroker.MqttDemoClient/Program.cs`

## Notes
- The shared core mapper currently owns the `items` segment via `ItemsSegment`; the client wrapper delegates topic generation/parsing to it.
- Current tests assert old examples such as `hornet/clients/device-client/items/Runtime/Device/Read/value`.
- Expected new examples:
  - `hornet/clients/device-client/Runtime/Device/Read/value`
  - `hornet/clients/device-client/Runtime/Device/Read/params/Unit`
  - `hornet/clients/demo-client/Demo/Temperature/value`

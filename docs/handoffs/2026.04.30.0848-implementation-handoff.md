# IMPLEMENTATION HANDOFF

## Goal
Hide the technical `Runtime.ItemBroker` prefix from user-facing target selection while keeping the internal broker runtime registry path stable.

User-facing BrokerWidget target examples should appear and persist as flat page-level paths such as:

`BrokerWidget1.DummyClient1.Edm1.Pressure`

The internal runtime registry may continue to use:

`Runtime.ItemBroker.BrokerWidget1.DummyClient1.Edm1.Pressure`

## Scope
Included:
- Make BrokerWidget attach targets appear flat in SignalWidget/ItemWidget target selection.
- Persist selected broker targets without the `Runtime.ItemBroker` prefix where practical.
- Resolve flat broker target paths back to internal runtime broker registry keys.
- Preserve existing saved values that already contain `Runtime.ItemBroker`.
- Keep attach option discovery under the existing editor discovery namespace unless a smaller direct change is required.

Not included:
- No MQTT protocol changes.
- No broad registry redesign.
- No removal of internal `Runtime.ItemBroker` storage unless proven safe.
- No unrelated target tree redesign.
- No new dependencies.

## Tasks
1. Inspect the current broker path flow in:
   - `BrokerClientControl.EnumerateAttachOptions()`
   - `BrokerClientControl.PublishRuntimeItems()`
   - `MainWindowViewModel.GetBrokerAttachItemOptions()`
   - target resolution in `PageItemModel`
   - `TargetTreeSelectionDialogWindow`
2. Keep internal broker snapshots registered under `Runtime.ItemBroker.<BrokerWidget>.<Client>.<ItemPath>`.
3. Change broker attach option values returned to target selection so they are user-facing flat paths:
   - from `Runtime.ItemBroker.BrokerWidget1.DummyClient1.Edm1.Pressure`
   - to `BrokerWidget1.DummyClient1.Edm1.Pressure`
4. Ensure `BrokerClientControl.ParseAttachedPaths()` accepts both old and new forms:
   - old: `Runtime.ItemBroker.BrokerWidget1.DummyClient1.Edm1.Pressure`
   - new: `BrokerWidget1.DummyClient1.Edm1.Pressure`
5. Add or update target resolution so a flat broker path resolves to the internal runtime key when the selected target belongs to a BrokerWidget on the same page.
6. Ensure the target tree displays only the flat hierarchy for broker targets:
   - `BrokerWidget1`
   - `DummyClient1`
   - `Edm1`
   - `Pressure`
7. Ensure saved layouts with existing `Runtime.ItemBroker...` values still load and resolve.
8. Add focused tests if a suitable test project exists:
   - flat broker attach option generation
   - legacy runtime broker path compatibility
   - flat target resolution to internal runtime broker key
9. Build:
   - `dotnet build src/HornetStudio.Editor/HornetStudio.Editor.csproj --no-restore -v:minimal`

## Notes
- `Runtime.ItemBroker` is useful as an internal global registry namespace.
- It should not be part of the normal user-facing target selection path.
- Prefer mapping flat user paths to runtime paths during resolution instead of duplicating broker snapshots under multiple registry keys.
- Avoid a purely cosmetic display-only change if it causes persisted paths or resolution to stay inconsistent.

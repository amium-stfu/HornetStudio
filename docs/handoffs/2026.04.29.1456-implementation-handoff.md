# IMPLEMENTATION HANDOFF

## Goal
Add recursive publishing of child items to `Amium.ItemBroker.Mqtt.Client` so publishing a parent item also publishes all nested subitems.

## Scope
Included:
- Extend MQTT client publish behavior to traverse `Amium.Item.Item` child items.
- Ensure snapshots include descendants.
- Ensure value publishing can handle descendants where appropriate.
- Add or update focused tests if a test project already covers MQTT client behavior.
- Update the demo client only if needed to validate the recursive behavior already prepared with `Raw` subitems.

Not included:
- MQTT subscription support.
- Broker store or in-memory broker recursive subscription changes.
- Public API redesign unless required for a small, explicit recursive option.
- New external dependencies.

## Tasks
1. Inspect `src/Amium.ItemBroker.Mqtt.Client/MqttItemBrokerClientSession.cs` and current MQTT client tests.
2. Add a private traversal helper that enumerates the root item and all descendants using each child's existing `Path` metadata.
3. Update `PublishItemAsync(...)` so snapshot and delta behavior applies to every item returned by the traversal.
4. Update `PublishValueAsync(ItemModel item, ...)` if expected behavior is to publish descendant values during recurring updates.
5. Preserve existing per-path state tracking in `_publishedStates` so child item deltas are evaluated independently.
6. Add or update tests verifying that an item with `Raw` child publishes topics for both the parent and child.
7. Build `src/Amium.ItemBroker.MqttDemoClient/Amium.ItemBroker.MqttDemoClient.csproj` with the repo `NuGet.Config`.

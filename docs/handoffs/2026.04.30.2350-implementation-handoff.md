# IMPLEMENTATION HANDOFF

## Goal
Change BrokerWidget MQTT publishing and write-back to use one shared flat topic per broker item so every client writes to the same MQTT topic, e.g. `hornet/Studio/Project/DefaultLayout/UdlClient1/m400/Set/Request`.

## Scope
Included:
- Use flat shared item topics for BrokerWidget published items and write-back.
- Change new BrokerWidget default broker paths from `HornetStudio.<LocalPath>` to `Studio.<LocalPath>`.
- Publish active local definitions to `BaseTopic/<BrokerPath as slash path>`.
- Subscribe writable active definitions to the same flat topic.
- Keep parameter topics under the item topic with `/params/<ParameterName>`.
- Use local value comparison and last-state suppression for echo protection.
- Keep protected parameter write blocking through `HostRegistryParameterPolicy`.
- Update tests and BrokerWidget documentation.

Not included:
- No migration of existing `HornetStudio...` broker paths.
- No legacy compatibility rewrite for old default paths.
- No UI mode switch for old/new topic mapping unless required by existing architecture.
- No broad redesign of remote attached item browsing beyond what is required for shared published topics.

## Tasks
1. Update MQTT topic mapping so BrokerWidget published item values map to `BaseTopic/<BrokerPath segments>`.
2. Update parameter mapping to `BaseTopic/<BrokerPath segments>/params/<ParameterName>`.
3. Keep reserved system/metadata topics separate if needed, preferably with underscore prefixes such as `_broker` or `_meta`.
4. Update `BrokerPublishedItemDefinitionCodec.BuildDefaultBrokerPath` to produce `Studio.<LocalPath>`.
5. Do not add migration from existing `HornetStudio.<LocalPath>` definitions.
6. Update publish logic so active definitions publish retained values/parameters to the flat shared topic.
7. Update write-back logic so writable active definitions subscribe to the exact same flat topic.
8. Remove source-client-id as the primary echo guard for shared topics.
9. Ignore incoming write-back messages when the target local value/parameter already equals the broker payload.
10. Preserve last-state suppression to avoid immediate self-publish loops.
11. Apply `Value` writes through `HostRegistries.Data.UpdateValue(LocalPath, value)`.
12. Apply parameter writes only through `HostRegistries.Data.TryUpdateUserParameter(...)`.
13. Log and ignore protected parameter writes without throwing disruptive UI exceptions.
14. Add/update tests for:
    - default broker path is `Studio.<LocalPath>`
    - value topic is `hornet/Studio/.../Request`
    - parameter topic is `hornet/Studio/.../Request/params/Unit`
    - all clients map to the same shared topic
    - writable shared-topic value updates write back locally
    - same-value self echoes are ignored
    - protected parameters are blocked
    - non-writable and inactive definitions are ignored
15. Update docs to describe the flat shared topic model and MQTT Explorer workflow.

## Technical Constraints
- Chat and planning are German; code, comments, XML documentation, UI text, file names, and identifiers are English.
- Code must follow existing project patterns and `.editorconfig`.
- Do not introduce external dependencies.
- `Writable = false` remains the safe default.
- Write-back must stay entry-scoped and exact; a writable parent must not implicitly make children writable unless that is already an explicit active writable definition.
- `Value` is writable only when the definition is active and `Writable = true`.
- Protected parameters must remain blocked by `HostRegistryParameterPolicy`.
- No automatic migration of old broker paths.
- Public classes or public members changed during implementation need XML documentation.

## Notes
- Preferred visible MQTT Explorer shape: `hornet/Studio/Project/DefaultLayout/UdlClient1/m400/Set/Request`.
- The MQTT `BaseTopic` is `hornet`; the BrokerPath should start with `Studio`, not `HornetStudio`, to avoid `hornet/HornetStudio/...`.
- Existing old definitions can be deleted/recreated or manually edited during development.
- Shared topics mean source identity is no longer encoded in the value topic; echo protection must be based on equality and local state.

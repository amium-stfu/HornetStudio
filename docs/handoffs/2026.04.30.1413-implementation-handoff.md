# IMPLEMENTATION HANDOFF

## Goal
Clean up the BrokerWidget implementation by removing unnecessary diagnostics, reducing duplicated helper code, and improving exception safety around MQTT connect/disconnect and runtime item publishing.

## Scope
Included:
- BrokerWidget editor control cleanup.
- Host-side BrokerWidget client safety improvements.
- Focused helper extraction or centralization only where it removes current duplication.
- Focused tests or existing test updates for changed behavior.

Not included:
- Redesigning the ItemBroker protocol.
- Replacing MQTTnet or adding new dependencies.
- Large UI redesign of the BrokerWidget.
- Changing public APIs unless required for snapshot-safe access.

## Tasks
1. Remove or downgrade production-noisy BrokerWidget diagnostics:
   - Remove `Console.WriteLine` calls from `BrokerClientControl`.
   - Remove `LogAttachDiagnostics` or guard it behind debug/trace-level logging.
   - Reduce `HostItemBrokerClient` and MQTT receive diagnostics to meaningful warnings/errors or opt-in debug output.
2. Make BrokerWidget connect/disconnect exception-safe:
   - Avoid blocking `.GetAwaiter().GetResult()` on UI paths where practical.
   - Ensure event handlers are always unsubscribed on connect failure and disconnect.
   - Ensure failed `DisposeAsync` does not leave UI state or registry cleanup inconsistent.
3. Make remote item access snapshot-safe:
   - Avoid direct UI enumeration over mutable `ItemDictionary`.
   - Add a narrow snapshot method/property to `IHostItemBrokerClient` if needed.
   - Raise diagnostics/events outside locks where possible.
4. Centralize duplicated BrokerWidget path helpers:
   - Consolidate relative broker path logic currently duplicated in `BrokerClientControl`, `ItemExposureDialogWindow`, and `ItemExposureDefinition`.
   - Prefer `TargetPathHelper` or a small Broker-specific helper in the editor project.
5. Review runtime registry publishing:
   - Keep current AttachToUi behavior intact.
   - Keep `Project.{Folder}.{Widget}.Status.AttachOptions` behavior unless a small, safe provider-based replacement already exists.
   - Ensure stale runtime and attach-option items are removed on disconnect/detach/failure.
6. Add or update focused tests:
   - Validate path normalization/relative path behavior.
   - Validate connect failure cleanup where testable.
   - Validate runtime item publishing still uses `Runtime.ItemBroker.{Widget}.{Client}.{ItemPath}`.
7. Build and run affected tests:
   - `src/HornetStudio.Editor/HornetStudio.Editor.csproj`
   - `src/HornetStudio.Host.Tests/HornetStudio.Host.Tests.csproj`
   - Any existing editor tests affected by path behavior.

## Technical Constraints
- Chat responses must be German.
- Code, comments, XML documentation, UI text, file names, and identifiers must be English.
- Do not add new dependencies.
- Keep changes scoped to BrokerWidget, host Broker client, and directly related helpers/tests.
- Preserve existing architecture and `.editorconfig` formatting.
- Public methods/classes changed or added must have XML documentation.
- User-visible errors and validation messages must be English.
- Do not silently redesign AttachToUi or target selection behavior.

# IMPLEMENTATION HANDOFF

## Goal
Change `BrokerWidget` so MQTT broker items are discovered internally but only registered in `HostRegistries.Data` after explicit attach selection, matching the `UdlClient` AttachToUi behavior and reducing global registry load.

## Scope
Included:
- Keep MQTT connection and remote item discovery working.
- Build attach options from the broker client's internal remote item cache.
- Persist selected broker attach paths in `FolderItemModel.BrokerAttachedItemPaths`.
- Register only selected broker items in `HostRegistries.Data`.
- Remove stale registered broker items when they are no longer selected or the widget disconnects.

Not included:
- MQTT protocol changes.
- Broker write routing or ownership changes.
- Refactoring `UdlClient`.
- Large UI redesign beyond the Broker attach workflow.

## Tasks
1. Inspect the existing `UdlClientControl` attach flow and reuse the same conceptual pattern where practical.
2. Refactor `BrokerClientControl` so available attach options are derived from `_client.Items` instead of `HostRegistries.Data`.
3. Replace full `PublishRuntimeItems()` mirroring with selective synchronization based on `BrokerAttachedItemPaths`.
4. Add or adapt UI interaction for selecting attach options and updating `BrokerAttachedItemPaths`.
5. Ensure selected items are registered under `Runtime.ItemBroker.{WidgetName}.{RemoteClientId}.{ItemPath}`.
6. Ensure unselected, stale, or disconnected broker runtime items are removed from `HostRegistries.Data`.
7. Keep `HostItemBrokerClient` as an internal cache owner and avoid direct global registration there.
8. Add focused tests if existing host/editor test coverage can validate path selection or registry synchronization.
9. Build affected projects with the repository `NuGet.Config`.

## Technical Constraints
- Chat responses must be German.
- Code, comments, XML documentation, UI text, file names, and technical identifiers must be English.
- Public changed or added methods/classes must have XML documentation.
- Use existing project patterns and dependencies.
- Do not introduce new dependencies.
- Keep changes limited to BrokerWidget attach behavior and directly related persistence/UI plumbing.
- Runtime path format must stay `Runtime.ItemBroker.{WidgetName}.{RemoteClientId}.{ItemPath}`.
- `HostRegistries.Data` must contain only explicitly attached broker items, not every discovered MQTT item.

- The broker client may still receive and cache all subscribed MQTT items for discovery.
- The performance optimization is avoiding global registry upserts, indexing, and events for unattached items.
- The attach option list can show discovered paths without registering those paths globally.
- On disconnect, remove only runtime keys owned by the current Broker widget prefix.

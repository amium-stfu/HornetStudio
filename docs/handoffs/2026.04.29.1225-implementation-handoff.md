# IMPLEMENTATION HANDOFF

## Goal
Add item-based publish convenience methods to the ItemBroker client session and update the demo to use them.

## Scope
Included:
1. Add `PublishValueAsync(Item item, ...)`.
2. Add `PublishParameterAsync(Item item, string parameterName, ...)`.
3. Keep the existing string/path-based publish methods for compatibility.
4. Update the demo client to instantiate real demo `Item` objects and publish through the new item-based API.

Not included:
- Do not move broker/session state into `Amium.Item.Item`.
- Do not change the meaning of snapshots, value deltas, or parameter deltas.
- Do not introduce new dependencies.
- Do not add broader abstractions unless required for the two new methods.
- Do not keep or reintroduce `DemoItemDefinition`.
- Do not keep a demo-local publish handle/wrapper if the new client API makes it unnecessary.

## Tasks
1. Update `src/Amium.ItemBroker.Client/IItemBrokerClientSession.cs` with overloads:
   - `Task PublishValueAsync(Item item, string? correlationId = null, CancellationToken cancellationToken = default);`
   - `Task PublishParameterAsync(Item item, string parameterName, string? correlationId = null, CancellationToken cancellationToken = default);`
2. Implement the overloads in `src/Amium.ItemBroker.Client/ItemBrokerClientSession.cs`.
3. For `PublishValueAsync(Item item, ...)`, validate `item`, resolve path from `item.Path ?? item.Name`, and publish `item.Value`.
4. For `PublishParameterAsync(Item item, string parameterName, ...)`, validate `item` and `parameterName`, resolve path from `item.Path ?? item.Name`, read `item.Params[parameterName].Value`, and publish that parameter value.
5. Reuse existing path-based methods internally to avoid duplicate message construction.
6. Keep XML documentation complete for all new public interface members and implementation members as required by project rules.
7. Update `src/Amium.ItemBroker.DemoClient/Program.cs` so the demo directly creates two items:
   - `Item Temperature`
   - `Item Pressure`
8. Give the demo items stable full paths, for example:
   - `new Item("Temperature", 22.0, "Demo")`
   - `new Item("Pressure", 1012.0, "Demo")`
9. Set metadata directly on the items:
   - `Temperature.Params["Unit"].Value = "degC";`
   - `Temperature.Params["Kind"].Value = "Temperature";`
   - `Temperature.Params["Format"].Value = "0.0";`
   - `Pressure.Params["Unit"].Value = "hPa";`
   - `Pressure.Params["Kind"].Value = "Pressure";`
   - `Pressure.Params["Format"].Value = "0.0";`
10. Publish both full item snapshots once before the update loop with `PublishItemAsync(item, ...)`.
11. In the 10 Hz update loop, assign the new values to the item instances first:
   - `Temperature.Value = temperature;`
   - `Pressure.Value = pressure;`
12. Publish value updates through the new item-based overloads:
   - `await client.PublishValueAsync(Temperature, cancellationToken: cancellationToken);`
   - `await client.PublishValueAsync(Pressure, cancellationToken: cancellationToken);`
13. Ensure the update loop does not publish `Unit`, `Kind`, `Format`, or other parameters again.
14. Use `PublishParameterAsync(item, "CX", ...)` only where the demo intentionally changes and publishes an individual parameter. Do not add a fake `CX` parameter unless the demo needs one.
15. Add or update focused tests in `src/Amium.ItemBroker.Tests/Program.cs` for item-based value and parameter publishing.
16. Build the relevant projects.

## Technical Constraints
- Chat responses must be in German.
- Code, comments, XML documentation, UI text, file names, and technical identifiers must be in English.
- Use existing project patterns.
- Existing APIs must remain source-compatible.
- `Item` remains a data model and must not reference broker/session/transport services.
- `PublishValueAsync(Item item, ...)` must publish only the item's `Value`.
- `PublishParameterAsync(Item item, "CX", ...)` must publish only the named parameter.
- The demo's repeated value publishing must use the new item-based overloads, not manual `path`/`value` extraction.

## Relevant Files
- `src/Amium.ItemBroker.Client/IItemBrokerClientSession.cs`
- `src/Amium.ItemBroker.Client/ItemBrokerClientSession.cs`
- `src/Amium.ItemBroker.DemoClient/Program.cs`
- `src/Amium.ItemBroker.Tests/Program.cs`

## Notes
- The preferred call style is `await client.PublishValueAsync(item);`.
- The preferred parameter call style is `await client.PublishParameterAsync(item, "CX");`.
- Avoid `client.PublishParameterAsync(item["CX"])` because `item["CX"]` is a child item, not a parameter.
- Path resolution should match existing `PublishItemAsync`: prefer `item.Path`, then `item.Name`, otherwise throw a descriptive exception.
- The demo should show the desired final usage style clearly: instantiate items, change `Item.Value`, then call `client.PublishValueAsync(item)`.

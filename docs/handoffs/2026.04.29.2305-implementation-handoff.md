# IMPLEMENTATION HANDOFF

## Goal
Fix BrokerWidget host registration paths so selected broker items are registered under the actual runtime broker item path instead of under the editor attach-options path.

Expected registered item path example:

`Runtime.ItemBroker.BrokerWidget1.DummyClient1.Edm1.Pressure`

The persisted/selected attach identity may remain the full runtime broker item path, but it must not include `Status.AttachOptions`.

## Scope
Included:
- Correct BrokerWidget attach option publishing and runtime registration path separation.
- Ensure attach options are exposed for the editor without contaminating the registered runtime item path.
- Preserve selected BrokerWidget attachments across refreshes.
- Keep the visual attach dialog behavior from the previous handoff intact.
- Add or update focused tests for BrokerWidget path construction if an appropriate test project already covers this area.

Not included:
- No MQTT protocol changes.
- No broad registry redesign.
- No UI redesign beyond labels/path display if needed to reflect the corrected paths.
- No new external dependencies.

## Tasks
1. Inspect `BrokerClientControl` path flow:
   - attach option generation in `EnumerateAttachOptions()`,
   - attach option publishing in `PublishAttachOptionItems()`,
   - selected attachment matching and runtime publication in `PublishRuntimeItems()`,
   - attach option base path in `GetAttachOptionsBasePath()`.
2. Keep `Project.<FolderName>.<WidgetName>.Status.AttachOptions...` only as the editor discovery location.
3. Ensure the attach option value/title/suffix exposed to `BrokerAttachedItemPaths` is the broker runtime path:
   - `Runtime.ItemBroker.<WidgetName>.<ClientName>.<ItemPath>`.
4. Ensure `PublishRuntimeItems()` matches selected paths against runtime broker paths only.
5. Ensure `HostRegistries.Data.UpsertSnapshot()` receives runtime broker keys only:
   - correct: `Runtime.ItemBroker.BrokerWidget1.DummyClient1.Edm1.Pressure`,
   - incorrect: `Project.<Folder>.BrokerWidget1.Status.AttachOptions.Runtime.ItemBroker...`,
   - incorrect: `BrokerWidget1.Status.AttachOptions.Runtime.ItemBroker...`.
6. Review `MainWindowViewModel.GetBrokerAttachItemOptions()`:
   - it may read suffixes from the attach-options registry path,
   - it must return the actual runtime broker item paths, not paths prefixed by `Status.AttachOptions`.
7. Review `EditorDialogField.GetBrokerDisplayParts()` after path correction:
   - display name should remain compact, e.g. `Pressure`,
   - source should remain useful, e.g. `DummyClient1 -> Edm1`.
8. Add/update tests if practical:
   - path extraction from attach option registry keys,
   - runtime path used for published attached item snapshots.
9. Build:
   - `dotnet build src/HornetStudio.Editor/HornetStudio.Editor.csproj --no-restore -v:minimal`
   - run relevant existing host/editor tests if available.

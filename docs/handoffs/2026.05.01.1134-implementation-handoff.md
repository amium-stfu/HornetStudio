# IMPLEMENTATION HANDOFF

## Goal
Use `Studio` as the BrokerWidget published item broker path prefix for new definitions and documentation, without migrating existing saved `HornetStudio` broker paths.

## Scope
Included:
- Change the default BrokerWidget broker path prefix for new published item definitions to `Studio`.
- Keep broker paths flat: `Studio.<LocalRegistryKey>`.
- Update examples, YAML sample, and documentation to use `Studio`.
- Add tests for the new default prefix.

Not included:
- No migration from `HornetStudio.*` to `Studio.*`.
- No automatic rewrite of existing saved layouts.
- No write-back behavior changes.
- No recursive write-back behavior changes.
- No MQTT topic mapping redesign.

## Tasks
1. Update `BrokerPublishedItemDefinitionCodec.BuildDefaultBrokerPath(...)` so new defaults are `Studio.<LocalPath>`.
2. Ensure `BrokerPublishedItemDefinitionCodec.CreateDefault(...)` uses the new `Studio` prefix through the default builder.
3. Keep parsed existing `BrokerPath` values unchanged, including old `HornetStudio.*` values.
4. Update BrokerWidget YAML sample to use `Studio.<LocalRegistryKey>`.
5. Update BrokerWidget documentation/help/descriptions to use `Studio` in examples.
6. Search for BrokerWidget publishing examples that still show `HornetStudio.<LocalPath>` and update only examples/default descriptions.
7. Add or update tests:
   - new default path is `Studio.<LocalPath>`
   - existing explicit `Studio.*` path remains unchanged
   - existing explicit `HornetStudio.*` path remains unchanged because no migration is desired

## Technical Constraints
- Chat and planning are German; code, comments, XML docs, UI text, file names, and identifiers are English.
- Do not migrate or rewrite existing saved `HornetStudio.*` broker paths.
- Do not add a `Projects` segment.
- Keep broker paths as flat as possible.
- Do not add dependencies.
- Public classes or public members changed during implementation need XML documentation.

## Notes
- Existing layouts with `HornetStudio.*` broker paths will continue to require MQTT writes to the old topic unless the user reselects/recreates the publish item or manually changes the saved broker path.
- The expected MQTT topic for a default local path like `Project.DefaultLayout.UdlClient1.m400.Set.Request` is `hornet/Studio/Project/DefaultLayout/UdlClient1/m400/Set/Request`.

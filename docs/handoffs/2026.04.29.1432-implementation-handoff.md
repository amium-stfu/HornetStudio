# IMPLEMENTATION HANDOFF

## Goal
Rename and reorganize the MQTT client project so the ItemBroker structure follows the transport-first layout:

- `Amium.ItemBroker`
- `Amium.ItemBroker.Client`
- `Amium.ItemBroker.Mqtt`
- `Amium.ItemBroker.Mqtt.Client`
- `Amium.ItemBroker.MqttDemoClient`
- `Amium.ItemBroker.Service`
- `Amium.ItemBroker.Tests`

## Scope
Included:
- Rename `src/Amium.ItemBroker.Client.Mqtt` to `src/Amium.ItemBroker.Mqtt.Client`.
- Rename the project file accordingly.
- Update solution/project references.
- Update namespaces from `Amium.ItemBroker.Client.Mqtt` to `Amium.ItemBroker.Mqtt.Client`.
- Update package metadata, documentation, and tests only where directly affected.

Not included:
- No behavioral changes to MQTT client or broker logic.
- No restructuring of `Amium.ItemBroker.Service`.
- No new dependencies.
- No unrelated refactoring.

## Tasks
1. Inspect references to `Amium.ItemBroker.Client.Mqtt` in solution files, project files, source code, tests, documentation, and handoffs.
2. Rename the folder to `src/Amium.ItemBroker.Mqtt.Client`.
3. Rename the `.csproj` to `Amium.ItemBroker.Mqtt.Client.csproj`.
4. Update the `.sln` entry and all `ProjectReference` paths.
5. Update namespaces and `using` statements from `Amium.ItemBroker.Client.Mqtt` to `Amium.ItemBroker.Mqtt.Client`.
6. Update assembly/package metadata in the renamed project if it contains old names.
7. Update directly affected docs such as `README.md`, `CHANGELOG.md`, or relevant project docs if they mention the old project name.
8. Build the affected projects, preferably the full solution.
9. Run relevant tests if available.

- Code, comments, XML documentation, UI text, file names, and technical identifiers must be in English.

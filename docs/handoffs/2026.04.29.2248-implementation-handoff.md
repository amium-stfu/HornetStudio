# IMPLEMENTATION HANDOFF

## Goal
Make the BrokerWidget `Attach Items` dialog visually calm and aligned so attachable broker items appear as clean rows, for example:

`[ ] Pressure`
`[ ] Pressure.Raw`
`[ ] Temperature`
`[ ] Temperature.Raw`

## Scope
Included:
- Improve only the BrokerWidget attach-list presentation in the existing `Attach Items` dialog.
- Keep full persisted attach paths unchanged.
- Keep missing saved attachment removal behavior.
- Keep non-BrokerWidget attach editors unchanged unless shared layout bindings require a minimal compatible adjustment.

Not included:
- No MQTT behavior changes.
- No registry path redesign.
- No new dependencies.
- No broad dialog redesign outside row alignment and compact BrokerWidget display.

## Tasks
1. Inspect the current BrokerWidget rows created by `EditorDialogField.RebuildBrokerAttachEntries()`.
2. Decide whether BrokerWidget should keep group rows visible or render only leaf rows.
3. Prefer a compact leaf-row display for BrokerWidget:
   - Name column shows the item path relative to the remote client and parent device, such as `Pressure` and `Pressure.Raw`.
   - Source column may show the stable source context, such as `DummyClient1 -> Edm1`, if still useful.
   - Do not repeat standalone group labels between selectable rows.
4. Update the row layout in `AttachItemsEditorDialogWindow.axaml` so every visible row uses fixed columns:
   - checkbox column has a fixed width and is present as spacing even for rows without a checkbox,
   - name column starts at the same x-position for every selectable row,
   - source column starts at the same x-position for every selectable row,
   - remove button stays in a stable trailing column.
5. If group rows remain visible, render them as full-width section headers that do not disturb checkbox/name/source alignment.
6. Ensure long names and source paths trim instead of pushing columns around.
7. Preserve `ApplyAttachItemEntries()` serialization so only selected non-group, non-removed full paths are saved.
8. Build `src/HornetStudio.Editor/HornetStudio.Editor.csproj`.

## Technical Constraints
- Chat responses must be in German.
- Code, comments, XML documentation, UI text, file names, and technical identifiers must be in English.
- User-visible UI text remains English.
- Use existing Avalonia patterns and current `AttachItemEditorRow` properties where possible.
- Avoid new abstractions unless the existing row model cannot express the required layout clearly.
- Public classes and public methods changed during implementation need XML documentation.

## Notes
- The current visual unrest is caused by mixed group rows and leaf rows sharing a flat grid where checkbox visibility and indentation change the apparent row start.
- The safest UX target is a stable table-like list for attachable leaves.
- For the shown example, the final selectable labels should scan vertically as `Pressure`, `Pressure.Raw`, `Temperature`, `Temperature.Raw`.

# TODO

## Title
Support plain MQTT value topics.

## Problem
External MQTT clients cannot publish a new Item value with a simple `topic = payload` message unless they use the existing structured Item topic format.

## Impact
Simple MQTT integrations require knowledge of the internal Item topic mapping, which makes interoperability harder for lightweight clients and manual MQTT tools.

## Suggested Fix
Add an optional MQTT server compatibility mode that interprets a configured plain topic prefix as Item value writes. The MQTT topic should map to the Item path and the payload should map to the Item `read` value. Keep the existing snapshot and property topic format unchanged.

## Priority
Medium

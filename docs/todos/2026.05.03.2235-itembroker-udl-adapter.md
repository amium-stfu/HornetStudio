# TODO

## Title
Evaluate ItemBroker UDL adapter.

## Problem
The current UDL client flow and ItemBroker flow are still separate runtime islands. UDL items can appear in HornetStudio, and ItemBroker can distribute items over transports such as MQTT, but there is no dedicated UDL-to-ItemBroker adapter boundary yet.

## Impact
A shared ItemBroker-based runtime model could reduce special-case paths, make UDL items available through the same broker ownership/write model, and allow MQTT or future transports to expose UDL data without UDL-specific integrations.

## Suggested Fix
Design `Amium.Item.Server.Udl` as an adapter that publishes `Amium.UdlClient` items into `Amium.Item.Server` and maps broker write requests back to UDL request/write items. Prefer the generic `Amium.Item.Client` for consumers unless a concrete UDL-specific client API becomes necessary.

## Priority
Medium

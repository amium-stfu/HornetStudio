# TODO

## Title
Simplify MQTT item transport with performance focus.

## Problem
The current MQTT item client/server implementation preserves important item broker behavior, but several hot-path classes combine transport handling, item protocol mapping, payload conversion, diagnostics, remote tree reconstruction, health handling, and mutation protection.

## Impact
This makes the implementation harder to understand and can add unnecessary per-message cost through repeated path normalization, payload parsing, dictionary work, diagnostics, and full item cloning.

## Suggested Fix
Analyze and restructure the MQTT item path around a lean hot path:

- Keep existing MQTT item broker functionality.
- Separate raw MQTT transport from item protocol mapping and broker bridging.
- Centralize payload parse/format behavior.
- Avoid full item cloning in change notifications where delta data is sufficient.
- Keep diagnostics, health publishing, and remote tree snapshots outside the critical message path where possible.
- Measure or inspect allocation-heavy paths before changing behavior.

## Priority
High

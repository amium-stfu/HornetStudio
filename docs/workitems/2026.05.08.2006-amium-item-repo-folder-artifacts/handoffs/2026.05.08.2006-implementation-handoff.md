# IMPLEMENTATION HANDOFF

## Goal
Move the Amium.Item codebase into a dedicated top-level `amium_item/` folder that can later become its own GitHub repository, and change HornetStudio integration so HornetStudio consumes built Amium.Item DLL/PDB artifacts instead of owning the Amium.Item projects directly.

The target split is:

- `amium_item/` contains the future standalone Amium.Item repository content.
- `artifacts/amium_item/<configuration>/` contains built DLL/PDB outputs for HornetStudio consumption.
- `src/` remains HornetStudio-owned source code.
- `HornetStudio.sln` builds HornetStudio against Amium.Item artifacts, not against Amium.Item source projects.

## Scope
Included:

- Move Amium.Item production libraries, tests, samples, tools, and documentation into `amium_item/`.
- Move `Amium.Item.sln` into `amium_item/`.
- Keep the production assemblies:
  - `Amium.Item.dll`
  - `Amium.Item.Server.dll`
  - `Amium.Item.Client.dll`
  - `Amium.Item.Mesh.dll`
- Configure Amium.Item builds to copy required DLL/PDB artifacts to `artifacts/amium_item/debug/` and `artifacts/amium_item/release/`.
- Update HornetStudio projects to reference Amium.Item assemblies via DLL references with `HintPath` entries under `artifacts/amium_item/<configuration>/`.
- Update the build routine so Amium.Item is built before HornetStudio.
- Preserve `Amium.Item.sln` as a standalone-buildable solution inside `amium_item/`.
- Preserve `HornetStudio.sln` as a standalone HornetStudio solution that expects Amium.Item artifacts to exist.
- Update documentation for the new folder split and build order.

Explicitly not included:

- Publishing NuGet packages.
- Creating GitHub repositories.
- Creating release versions or Git tags.
- Implementing source package/source link support.
- Removing Amium.Item namespaces from source compatibility.
- Changing runtime behavior of Amium.Item, HornetStudio, MQTT, server, client, or mesh.

- Artifact paths are generated outputs and should be treated as build artifacts, not source.

- HornetStudio should reference Amium.Item DLLs plus PDBs from `artifacts/amium_item/<configuration>/`.

## Tasks
1. Inventory every Amium.Item-related project, solution entry, project reference, documentation file, tool, sample, and test.
2. Define the exact `amium_item/` folder layout before moving files.
3. Move Amium.Item source projects into `amium_item/src/`.
4. Move Amium.Item tests into `amium_item/tests/`.
5. Move Amium.Item samples into `amium_item/samples/`.
6. Move Amium.Item tools into `amium_item/tools/`, except tools that depend directly on HornetStudio runtime APIs and must remain in the HornetStudio area.
7. Move Amium.Item-specific documentation into `amium_item/docs/`.
8. Move `Amium.Item.sln` into `amium_item/Amium.Item.sln` and update all relative paths.
9. Decide whether `Directory.Build.props`, `NuGet.Config`, and `global.json` should be copied into `amium_item/` for future standalone repository readiness, or kept root-only with documented dependency.
10. Configure Amium.Item production library builds to copy DLL and PDB outputs into `../artifacts/amium_item/<configuration>/` relative to repository root, or an equivalent stable path.
11. Ensure artifact output includes all runtime dependencies required by HornetStudio if raw assembly references need them at runtime.
12. Replace HornetStudio-side `ProjectReference` entries to Amium.Item projects with assembly `Reference` entries using configuration-aware `HintPath` values.
13. Update HornetStudio tests and tool projects that currently reference Amium.Item source projects.
14. Remove Amium.Item source projects from `HornetStudio.sln`.
15. Keep only HornetStudio-owned projects in `HornetStudio.sln`, unless a project intentionally remains as a local tool requiring both HornetStudio and Amium.Item artifacts.
16. Add or update build scripts/docs so the required order is:
    1. Build `amium_item/Amium.Item.sln`.
    2. Copy/generate artifacts.
    3. Build `HornetStudio.sln`.
17. Update `.gitignore` if needed so generated artifacts are handled intentionally.
18. Build and verify both solutions.

9. Remove Amium.Item source projects from `HornetStudio.sln`.
10. Update any HornetStudio-owned tests/tools to use artifact references or move them under `amium_item/` if they are actually Amium.Item-owned.
11. Update docs and build instructions.
12. Build `HornetStudio.sln` after Amium.Item artifacts exist.
13. Run relevant tests.

## Technical Constraints
- Chat responses must be in German.
- Code, comments, XML documentation, UI text, file names, and technical identifiers must be in English.
- Newly created path-relevant names must use snake_case.
- Preserve existing C# namespaces and public API names unless a change is required by the move.
- Do not introduce new runtime dependencies.
- Do not create NuGet packages in this step.
- Do not change release versioning in this step.
- Do not manually check in generated DLL/PDB artifacts unless explicitly decided; generated artifacts should normally be reproducible build outputs.
- `Amium.Item.dll` must not become a monolithic all-in-one assembly.
- HornetStudio should consume only the Amium.Item assemblies it actually needs.

## Risks / Watchouts
- Raw DLL references are stricter than project references. Missing transitive runtime dependencies can surface only at runtime if not copied or referenced correctly.
- Configuration-aware `HintPath` handling can become fragile if Debug/Release output paths diverge.
- PDBs help debugging, but source-level debugging may still require source files at matching paths unless SourceLink is added later.
- Tools currently under `tools/` may mix Amium.Item and HornetStudio dependencies. Do not move HornetStudio-coupled tools blindly.
- `Directory.Build.props` currently provides shared version metadata. A future standalone `amium_item/` repo likely needs its own copy or replacement.
- `NuGet.Config` and `global.json` may also need to be duplicated or moved later for standalone builds.
- Build artifacts should normally be ignored by Git. If committed local DLLs are desired temporarily, document that explicitly.

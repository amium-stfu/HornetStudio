# DEBUG REPORT

## Problem
`ItemProperty.LastUpdate` was renamed or partially renamed to `Epoch`, which caused compile errors in `ItemExtension.cs`. HornetStudio also no longer starts after the Amium.Item changes.

## Expected Behavior
Amium.Item and HornetStudio should compile against one consistent `ItemProperty` timestamp API. HornetStudio should load the same Amium.Item assembly version that matches the source/API expected by its projects.

## Actual Behavior
The IDE reports CS1061 errors in `amium_item/src/Amium.Item/ItemExtension.cs` because `ItemProperty.LastUpdate` is referenced while the type was expected to expose `Epoch` instead.

HornetStudio startup is likely affected by one or both of these issues:

- Amium.Item source/API and HornetStudio references are not in sync.
- HornetStudio consumes DLL artifacts from `artifacts/amium_item/debug/`, so stale or partially rebuilt artifacts can make runtime behavior differ from current source.

## Error Messages / Logs
CS1061:

- `ItemProperty` contains no definition for `LastUpdate`.
- Reported locations:
  - `amium_item/src/Amium.Item/ItemExtension.cs:77`
  - `amium_item/src/Amium.Item/ItemExtension.cs:154`

## Relevant Code
Current `ItemExtension.cs` still uses:

- `itemProperty.LastUpdate`
- `clonedItemProperty.LastUpdate = itemProperty.LastUpdate`

Current search also found `LastUpdate` usages in:

- `src/HornetStudio.Host/HostRegistries.cs`
- `src/HornetStudio.Host/UiFolderContext.cs`
- `src/HornetStudio.Host/Legacy/UiPageContext.cs`
- `amium_item/src/Amium.Item.Mesh/Controllers/mesh_demo_node_controller.cs`
- `amium_item/samples/amium_item_server_multimaster_demo/Controllers/DemoNodeController.cs`

Current `Item.cs` in the working tree contains both:

- public field `LastUpdate`
- public property `Epoch` as alias to `LastUpdate`

This means the repository is in an intermediate state: either the IDE saw a version where `LastUpdate` was already removed, or the source/artifacts/design-time build cache are not synchronized.

## Attempted Fixes
- No code fix was applied during this debug pass.
- Existing debug reports were checked; none existed in this workitem.

## Current Hypothesis
- The root cause of the CS1061 errors is an incomplete public API rename from `LastUpdate` to `Epoch`.
- A full rename requires updating every consumer of `LastUpdate`, including serialization/cloning, host registries, UI folder contexts, mesh demo code, samples, and tests.
- Keeping both `LastUpdate` and `Epoch` temporarily can make source compile, but it weakens the API and may hide stale artifact problems.
- HornetStudio likely fails to start because it references built Amium.Item artifacts instead of source projects. If those artifacts were built from an intermediate or incompatible API state, HornetStudio can compile or run against a different contract than the open source files suggest.

## Proposed Solution
Use one timestamp API name consistently.

Recommended short-term safe path:

1. Keep `LastUpdate` as a compatibility alias only if immediate downstream breakage must be avoided.
2. Introduce `Epoch` as the intended API.
3. Update all source references from `.LastUpdate` to `.Epoch`.
4. Keep JSON field compatibility deliberately: either continue serializing `"LastUpdate"` for existing payload compatibility or migrate with explicit backward-compatible deserialization.
5. Rebuild `amium_item/Amium.Item.sln` so `artifacts/amium_item/debug/` is refreshed.
6. Rebuild `HornetStudio.sln`.
7. If HornetStudio still fails to start, capture the actual runtime exception/log before changing more code.

Preferred clean path once consumers are updated:

1. Remove `LastUpdate`.
2. Keep only `Epoch`.
3. Update serialization payload field names only if this is an intentional compatibility break.

## Watchouts
- Renaming serialized JSON fields from `LastUpdate` to `Epoch` can break existing saved snapshots or broker payload consumers.
- This issue overlaps with the planned `read` invariant work because `value` fallback timestamp reads still exist in host and mesh code.
- Do not fix HornetStudio startup by editing random startup code until the Amium.Item artifact/source mismatch is eliminated.

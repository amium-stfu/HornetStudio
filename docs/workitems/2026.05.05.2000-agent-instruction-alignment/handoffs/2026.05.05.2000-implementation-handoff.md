# IMPLEMENTATION HANDOFF

## Goal
`AGENTS.md` als führende gemeinsame Regelquelle für Visual Studio, GitHub Copilot und VS Code + Codex etablieren und `.github/copilot-instructions.md` darauf abgestimmt verschlanken.

## Scope
Included:
- Bereinigung und Rollenklärung zwischen `AGENTS.md` und `.github/copilot-instructions.md`
- Reduktion redundanter oder konkurrierender Regeln
- Erhalt der Mode- und Workflow-Regeln in `AGENTS.md`
- Erhalt notwendiger Copilot-spezifischer Ergänzungen in `.github/copilot-instructions.md`

Excluded:
- Produktcodeänderungen
- Solution- oder Projektstrukturänderungen
- neue Tooling-Mechanismen außerhalb der beiden Regeldateien

## Starting Point
- `AGENTS.md` enthält aktuell allgemeine Regeln, Modes, Workflow, Workitems, Qualitäts- und Dokuvorgaben.
- `.github/copilot-instructions.md` enthält ebenfalls allgemeine Regeln und bereichsspezifische Spezialvorgaben.
- Der Nutzer arbeitet auch mit VS Code + Codex, wobei dort `AGENTS.md` berücksichtigt wird.
- Daher darf `AGENTS.md` nicht zu einer rein optionalen Nebenquelle werden.

## Tasks
1. Beide Dateien in Regelkategorien aufteilen: allgemeine Verhaltensregeln, Workflow/Modes, Codequalität, Dokumentation, bereichsspezifische Spezialregeln.
2. Für jede Kategorie festlegen, welche Datei führend ist.
3. `AGENTS.md` so überarbeiten, dass sie die vollständige zentrale Regelbasis für den Cross-Tool-Workflow bleibt.
4. `.github/copilot-instructions.md` so überarbeiten, dass sie keine konkurrierenden Regeln mehr enthält und nur kompatible Ergänzungen oder Copilot-spezifische Hinweise behält.
5. Beide Dateien abschließend auf Widersprüche, Doppelungen und unklare Prioritäten prüfen.
6. Falls durch die Änderung das erwartete Verhalten oder die Projektkonventionen dokumentiert werden müssen, relevante Markdown-Dokumentation aktualisieren.

## File-Level Changes
- `AGENTS.md` -> zentrale, vollständige Regelquelle für Cross-Tool-Nutzung; allgemeine Regeln + Workflow + Modes + Workitems konsistent formulieren
- `.github/copilot-instructions.md` -> Copilot-spezifische Ergänzungen und bereichsspezifische Spezialregeln beibehalten, redundante allgemeine Regeln kürzen oder auf `AGENTS.md` abstimmen
- `CHANGELOG.md` -> nur aktualisieren, wenn die Repo-Konventionen solche Regeländerungen dort dokumentiert haben sollen

## Implementation Order
1. Regelkategorien aus beiden Dateien vergleichen und Überschneidungen markieren.
2. `AGENTS.md` zuerst finalisieren, da sie die führende Quelle sein soll.
3. Danach `.github/copilot-instructions.md` an `AGENTS.md` angleichen und auf Ergänzungsrolle reduzieren.
4. Abschließende Konsistenzprüfung beider Dateien durchführen.

## Technical Constraints
- `AGENTS.md` muss für VS Code + Codex vollständig brauchbar bleiben.
- Chat-Antworten bleiben auf Deutsch, Code- und Regeltexte in Dateien bleiben auf Englisch, sofern die bestehende Datei dies so vorgibt.
- Keine unnötigen Strukturänderungen außerhalb des angefragten Scopes.
- Bestehende bereichsspezifische Spezialregeln aus `.github/copilot-instructions.md` dürfen nicht versehentlich verloren gehen.

## Acceptance Criteria
- `AGENTS.md` beschreibt weiterhin vollständig den gewünschten Workflow für toolübergreifende Nutzung.
- `.github/copilot-instructions.md` enthält keine widersprüchlichen allgemeinen Regeln mehr.
- Die Rollen beider Dateien sind nach dem Lesen eindeutig erkennbar.
- Die für Copilot relevanten Spezialregeln bleiben erhalten.

## Verification
- Beide Dateien vollständig lesen und kategorisch gegeneinander prüfen.
- Manuell prüfen, ob identische Themen nur einmal führend geregelt sind.
- Prüfen, ob `AGENTS.md` ohne zusätzliche Annahmen für VS Code + Codex ausreicht.

## Out of Scope
- Änderungen an Source Code, Tests, Solution-Foldern oder Build-Konfiguration
- neue Regeldateien oder externe Konfigurationsmechanismen

## Risks / Watchouts
- Zu starke Kürzung von `.github/copilot-instructions.md` kann Copilot-spezifische Kontextsignale schwächen.
- Unterschiedlich formulierte Prioritätsregeln in beiden Dateien können erneut Mehrdeutigkeit erzeugen.
- Bereichsspezifische Spezialregeln dürfen nicht versehentlich aus dem Workspace entfernt werden.

## Relevant Files (if known)
- `AGENTS.md`
- `.github/copilot-instructions.md`
- `docs/workitems/2026.05.05.2000-agent-instruction-alignment/plans/2026.05.05.2000-plan.md`

## Notes
- Ziel ist keine theoretische Perfektion, sondern eine robuste und alltagstaugliche Regelstruktur für mehrere Clients.
- Wenn eine Regel nur für Copilot sinnvoll ist, sollte sie nicht die führende Cross-Tool-Regel in `AGENTS.md` ersetzen.

# IMPLEMENTATION HANDOFF

## Goal
Rename the primary application-facing MQTT client from `MqttRemoteItemClient` to `MqttItemClient` and add a simplified constructor optimized for normal user code.

The intended normal usage should become:

```csharp
await using var client = new MqttItemClient(
    clientId: "demo-client",
    host: "127.0.0.1",
    port: 1883,
    baseTopic: "hornet");
```

The existing options-based construction path should remain available:

```csharp
await using var client = new MqttItemClient(new MqttItemClientOptions { ... });
```

## Scope
Included:
- Rename `MqttRemoteItemClient` to `MqttItemClient`.
- Rename `MqttRemoteItemClient.cs` to `MqttItemClient.cs`.
- Add a simple public constructor to `MqttItemClient`.
- Keep `MqttItemClientOptions` for full configuration.
- Update active source consumers, tests, samples, tools, and current docs.
- Update the simple CLI demo so it demonstrates `MqttItemClient` instead of `MqttItemClientSession` if that can be done without changing demo behavior materially.
- Keep `MqttItemClientSession` as the lower-level concrete MQTT session for Mesh, tools, tests, and specialized direct publish/subscribe scenarios.

Not included:
- Do not keep a compatibility wrapper or alias named `MqttRemoteItemClient`.
- Do not rename `MqttItemClientSession` in this change.
- Do not make `MqttItemClientSession` internal in this change.
- Do not rename `MqttRemoteItemRegistry` unless required by the client rename.
- Do not change MQTT topic semantics, retained-state behavior, self-published path hiding, subscription behavior, write behavior, or reconnect behavior.
- Do not add a helper class named `Client`.
- Do not introduce transport-neutral abstractions.

## Starting Point
- MQTT client types now live directly under the `Amium.Item.Client` namespace.
- `MqttRemoteItemClient` is the current primary application-facing facade.
- `MqttItemClientSession` is still used directly by Mesh, tests, tools, and some samples.
- Current references to `MqttRemoteItemClient` exist in:
  - `src/HornetStudio.Host/HostItemBrokerClient.cs`
  - `tools/item_server_monitor/Hosting/MonitorRegistryBridge.cs`

- The CLI demo currently uses `MqttItemClientSession`, which makes the sample look like the normal user path is the low-level API.

# IMPLEMENTATION HANDOFF

## Goal
Slim down and clarify the `Amium.Item.Client` public MQTT client surface so the normal application-facing API is simple, while Mesh-specific MQTT orchestration is isolated from the generic client SDK.

## Scope
Included:
- Treat `MqttRemoteItemClient` as the primary simple public MQTT client facade.
- Reduce strategic importance of `IItemClientSession`; remove it only if local compile impact is small and all references are migrated.
- Keep `MqttItemClientSession` available as a low-level MQTT building block for now, because Mesh, samples, tests, and tools still use it directly.
- Move Mesh-specific session orchestration behind a Mesh-owned client/helper so Mesh no longer shapes the generic client API directly.
- Remove or replace the client-side `MqttItemTopicMapper` wrapper if source compatibility is not needed inside this repository.
- Consolidate duplicated MQTT payload parsing/formatting and shared MQTT item constants where it reduces duplication without introducing a broad abstraction.

Not included:
- Do not introduce support for named pipes or another future transport.
- Do not split `Amium.Item.Client` into a new MQTT project.
- Do not redesign MQTT topic semantics.
- Do not change retained-state policy or echo-hiding behavior unless required by the API cleanup.
- Do not optimize batching or high-throughput publishing beyond cleanup needed for the API shape.

## Starting Point
- `HornetStudio` production code uses `MqttRemoteItemClient` through `HostItemBrokerClient`.
- `HornetStudio` does not rely on `IItemClientSession`.
- `Amium.Item.Mesh` directly owns multiple `MqttItemClientSession` instances for publisher, mirror, and peer writer roles.
- `Amium.Item.Mesh` also owns `MqttRemoteItemClient` observers.
- `MqttItemTopicMapper` in `Amium.Item.Client.Mqtt` is a thin wrapper around the shared/server MQTT mapper and is mainly used by tests/tools.
- `MqttRemoteItemRegistry` is used indirectly through `MqttItemClientSession.RemoteItems` and `MqttRemoteItemClient.GetRemoteItemSnapshots()`.
- `ItemClientSession` has an unused `_clock` field and constructor parameter.

## Technical Constraints
- Chat responses must be German; code, comments, XML documentation, UI text, file names, and identifiers must be English.
- Existing libraries and patterns must be preferred.
- Do not add new dependencies.
- Keep path-relevant new file names in snake_case.
- Public classes and public methods must have XML documentation.
- Avoid broad new abstractions; add only small internal helpers with clear duplication/performance benefit.
- Preserve `MqttRemoteItemClient` as the simple external API for `HornetStudio`.
- Do not make Mesh experimental requirements dictate the generic client API.

## Acceptance Criteria
- `HornetStudio.Host` still compiles and continues to use `MqttRemoteItemClient` as its MQTT ItemBroker facade.
- `Amium.Item.Mesh` no longer directly spreads raw MQTT session orchestration throughout `MeshDemoNodeController`; Mesh-specific behavior is isolated behind an internal Mesh type.
- `MqttRemoteItemClient` remains the primary documented public MQTT client facade.
- `IItemClientSession` is removed or no longer positioned as the future transport abstraction.
- The client-side `MqttItemTopicMapper` wrapper is removed/replaced, or explicitly retained only with a documented compatibility reason.
- Duplicate MQTT payload parsing/constants are reduced without adding a general transport abstraction.
- Existing behavior for remote item snapshots, self-published path hiding, publish/update, and subscribe is preserved.

## Out of Scope
- Named pipe support.
- New transport abstraction.
- New packages.
- MQTT batching or queue redesign.
- Moving all MQTT code into Mesh.
- Removing `MqttItemClientSession` while Mesh, tests, samples, and tools still rely on it.

## Risks / Watchouts
- `MqttRemoteItemClient` hides self-published paths intentionally; breaking this can regress `HostItemBrokerClient` and Mesh observer behavior.
- Mesh currently uses separate writer sessions because using `MqttRemoteItemClient` for writes previously caused observer-side hiding problems.
- `MqttItemClientSession.RemoteItems` is used by tests and host test receivers; making the registry private would require a replacement inspection API.
- Removing public APIs can break samples/tools even if production `HornetStudio` is unaffected.
- The shared mapper namespace can be confusing because server and client code currently import mapper names through aliases.

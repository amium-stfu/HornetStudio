# DEBUG REPORT

## Problem
The monitor tree hover highlight still jumps or flickers during periodic refresh after moving the style target from the template `ContentPresenter` to the row `Border`.

## Expected Behavior
Hover highlighting stays on the same row while values refresh, until the pointer moves to another row or leaves the tree.

## Actual Behavior
The same jump behavior remains, so the issue is not only the specific XAML selector used for the built-in pointer-over visual.

## Error Messages / Logs
No build or runtime exception was identified.

## Relevant Code
- `src/Item.Server.Monitor/MainWindow.axaml`
- `src/Item.Server.Monitor/MainWindow.axaml.cs`
- `src/Item.Server.Monitor/ViewModels/MonitorTreeRowViewModel.cs`

## Attempted Fixes
- Row view models are reused by path -> selection and row identity are stable.
- Virtualization is disabled with a `StackPanel` items panel -> container recycling is unlikely to be the remaining cause.
- Styling `Border.monitor-tree-row` from `ListBoxItem:pointerover` -> did not change the behavior.
- Current fix: remove dependency on Avalonia row `:pointerover` state for the monitor tree and store a custom `IsHovered` flag on the row view model. `MainWindow` updates this flag only from `ListBox.PointerMoved` and clears it on `PointerExited`.

## Current Hypothesis
- Refresh-driven layout or pointer re-evaluation still toggles Avalonia's built-in `:pointerover` state even when the visual target changes.
- A custom hover state avoids refresh-triggered hover recalculation because automatic updates change row content but do not change `IsHovered`.
- If the highlight still jumps after this fix, the next likely cause is actual visible-row order or vertical layout shifting during refresh, not pointer styling.

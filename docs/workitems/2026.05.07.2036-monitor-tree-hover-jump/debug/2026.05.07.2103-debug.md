# DEBUG REPORT

## Problem
The monitor tree hover highlight still jumps or flickers during periodic refresh after replacing the XAML hover selector with a custom row `IsHovered` state.

## Expected Behavior
The visual highlight stays attached to the row under the pointer while the monitor refreshes values.

## Actual Behavior
The same visual jump remains, which means the visible effect is not only caused by the custom row border style.

## Error Messages / Logs
No build or runtime exception was identified.

## Relevant Code
- `src/Item.Server.Monitor/MainWindow.axaml`
- `src/Item.Server.Monitor/MainWindow.axaml.cs`
- `src/Item.Server.Monitor/ViewModels/MainWindowViewModel.cs`
- `src/Item.Server.Monitor/ViewModels/MonitorTreeRowViewModel.cs`

## Attempted Fixes
- Row identity reuse by path -> already stable.
- Non-virtualized `StackPanel` items panel -> already in place.
- Hover style moved away from template `ContentPresenter` -> no behavioral change.
- Custom `IsHovered` row state while keeping `ListBox` -> no behavioral change.
- Current fix: replace the monitor tree `ListBox` with `ScrollViewer` + `ItemsControl`, so no native `ListBoxItem` container or Fluent `ListBoxItem` hover/selection template participates. Hover and selection are now both row-owned classes.

## Current Hypothesis
- The remaining visual jump was still coming from native `ListBoxItem` template state or container behavior, not from the row view model.
- Removing `ListBoxItem` entirely is the narrowest reliable way to disable built-in hover and selection visuals while preserving the existing flat row model.
- If this still jumps, the remaining likely root cause is actual row movement caused by visible row additions/removals or ordering changes during refresh.

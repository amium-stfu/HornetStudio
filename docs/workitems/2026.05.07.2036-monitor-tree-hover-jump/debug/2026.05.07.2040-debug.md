# DEBUG REPORT

## Problem
The monitor tree hover highlight jumps or flickers during periodic refresh while the pointer rests over a visible row.

## Expected Behavior
The row under the pointer keeps a stable hover background while item values refresh.

## Actual Behavior
The hover background can jump or flicker on every refresh even though selection is already preserved.

## Error Messages / Logs
No build or runtime exception was identified. The issue is visual behavior during live refresh.

## Relevant Code
- `src/Item.Server.Monitor/MainWindow.axaml`
- `src/Item.Server.Monitor/ViewModels/MainWindowViewModel.cs`
- `src/Item.Server.Monitor/ViewModels/MonitorTreeRowViewModel.cs`
- `src/Item.Server.Monitor/Monitoring/MonitorTreeStore.cs`

## Attempted Fixes
- Reuse visible row view models by path -> selection identity is stable and row instances are no longer rebuilt on each refresh.
- Disable list virtualization with a `StackPanel` items panel -> container recycling is less likely to be the cause.
- Move hover styling into the row template -> still leaves hover tied to a child visual that is affected by refresh-driven content invalidation.
- Current fix: bind the visible row background to the stable `ListBoxItem:pointerover` and `ListBoxItem:selected` state, targeting `Border.monitor-tree-row`, and remove the direct `Border.monitor-tree-row:pointerover` dependency.

## Current Hypothesis
- The remaining hover jump is caused by hover styling being applied through unstable template/child visual state while refreshes update bound row content.
- `MainWindowViewModel.SyncVisibleRows` already preserves row identity by `Path`, so the remaining issue is more likely styling/pointer state than collection replacement.
- Styling the row border from the owning `ListBoxItem` keeps the visual hover state attached to the item container instead of the refreshed child controls.

# DEBUG REPORT

## Problem
The current `UdlClient` attach-sections implementation introduced two UI regressions:

1. The `Attached Items` section no longer offers the `Edit` entry point that opened the exposure configuration.
2. The header now exposes additional action buttons besides the `Connected` button.

## Expected Behavior
- `Attached Items` should still provide an `Edit` action for the exposure menu in addition to attachment management.
- The header should match the intended compact control set and not expose the extra action buttons if only `Connected` should remain interactive.

## Actual Behavior
- Each attached row exposes only a single `Detach` action.
- The header now renders three interactive controls: `Connected`, `AttachToUi`, and `Modules`.

## Error Messages / Logs
No runtime exception was identified. This is a behavioral/UI regression caused by the current refactor.

## Relevant Code
- `src/HornetStudio.Editor/Widgets/UdlClient/UdlClientControl.axaml`
  - Header action grid now contains `AttachToUi` and `Modules` buttons.
  - `Attached Items` row template contains only one action button bound to `OnDetachAttachedItemClicked`.
- `src/HornetStudio.Editor/Widgets/UdlClient/UdlClientControl.axaml.cs`
  - `RebuildAttachSectionRows()` creates `UdlClientAttachSectionRow` with a single `ActionText` / `CanExecuteAction` model.
  - New header handlers `OnHeaderAttachClicked()` and `OnHeaderConfigureModulesClicked()` make both extra header buttons interactive.

## Attempted Fixes
- Restored the `Edit` button in `Attached Items` and removed extra header buttons.
- Result: `Edit` is visible again, but remains disabled for current attached rows.

## Current Hypothesis (optional)
- Regression 1: the body refactor simplified attached rows to a one-action row model (`Detach` only), so the earlier exposure-edit entry point was dropped during the layout rewrite.
- Regression 2: the plan/handoff explicitly allowed an attach-oriented header action, and the implementation expanded that into two header buttons. If the desired target is "only `Connected` remains as a header button", then the current implementation followed the wrong interaction model.
- Current inactive-`Edit` root cause: `CanEdit` depends on `ModuleName`, but `ModuleName` is currently derived by `TryGetRuntimeModuleName()` only when the attached path has at least two segments.
- `GetAttachOptions()` intentionally filters to root attach paths via `IsRootAttachPath()`, which means attached values are commonly single-segment module roots.
- Because attached values like `ModuleA` have only one segment, `TryGetRuntimeModuleName()` returns `false`, `ModuleName` becomes empty, and `CanEdit` disables the button.
- Current `Edit` flicker root cause: `PublishClientItems()` rebuilds `AttachedItems` unconditionally on every runtime publish pass via `RebuildAttachSectionRows()`.
- `PublishClientItems()` runs from `MonitorLoopAsync()` whenever `_clientItemsDirty` is set by `OnClientFrameReceived()`. The monitor loop polls every 250 ms, so active traffic can recreate the attached row controls several times per second.
- This recreates the `Attached Items` row template and its buttons repeatedly, which matches the visible button flicker.
- The 2-second `ScheduleAttachedItemsRefresh()` timer adds additional rebuilds, but the dominant visual churn comes from the unconditional body-row rebuild in `PublishClientItems()`.
- Current property-panel cleanup root cause: the UDL widget still explicitly registers both structured editors in `MainWindowViewModel.BuildBindingSectionsForItem()` for `ControlKind.UdlClientControl`.
- `src/HornetStudio.Editor/ViewModels/MainWindowViewModel.cs:4601` adds `BindUdlModuleExposureList("UdlModuleExposureDefinitions", "ModuleExposures", ...)`.
- `src/HornetStudio.Editor/ViewModels/MainWindowViewModel.cs:4602` adds `BindAttachItemList("UdlAttachedItemPaths", "AttachToUi", ...)`.
- These are editor-only bindings; removing them from the property section does not remove persistence or runtime behavior because the underlying model fields and serialization paths remain in place.

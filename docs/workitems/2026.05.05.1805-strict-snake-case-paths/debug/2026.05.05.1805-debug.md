# DEBUG REPORT

## Problem
MQTT broker files do not compile after the strict snake case path changes because broker contract and helper types are unresolved.

## Expected Behavior
The MQTT projects should compile against the broker contracts and helpers from `Amium.Item.Server`.

## Actual Behavior
The compiler reports missing types such as `IItemServerTransport`, `IItemServerClient`, `IItemServer`, `IItemSubscription`, `ItemServerMessage`, and `ItemSnapshotMessage`.
After fixing that file, the build reports unresolved `ItemServerHealthPaths` and `ItemServerPath` in `MqttTopicMapping.cs`.
After building the full solution, the MQTT client project reports unresolved `ItemServerValueCoercion` in `MqttRemoteItemRegistry.cs`.
After fixing the MQTT projects, the host project reports unresolved framework namespaces because `System.Runtime` usings were changed to `System.runtime`.
After fixing framework namespaces, the host project reports incorrectly cased own identifiers: `result.project` and `EnhancedSignalruntime`.
After the host project builds, the editor project reports Avalonia `x:Class` lookup errors because `HornetStudio` namespaces were changed to `Hornetstudio`, plus additional `System.runtime` and PascalCase runtime identifier issues.
After those fixes, the remaining editor errors are incorrectly cased `EnhancedSignalRow.Runtime` references.

## Error Messages / Logs
CS0246 for broker contract and message types in `src/Amium.Item.Server.Mqtt/MqttItemServerAdapter.cs`.
CS0103 for broker path helpers in `src/Amium.Item.Server.Mqtt/MqttTopicMapping.cs`.
CS0103 for broker value coercion in `src/Amium.Item.Client.Mqtt/MqttRemoteItemRegistry.cs`.
CS0234 and CS0246 for framework namespaces and attributes in `src/HornetStudio.Host`.
CS1061 for `ProjectBuildResult.project` and CS0103 for `EnhancedSignalruntime`.
AXN0001 for Avalonia XAML code-behind type lookup, CS0234/CS0246 for `System.runtime`, and CS0103 for `ApplicationExplorerruntime`/`ownedBrokerruntime`.
CS0103/CS1061 for `runtime` instead of `Runtime` in `EnhancedSignalsControl.axaml.cs`.

## Relevant Code
`MqttItemServerAdapter.cs` uses broker contracts from the `Amium.Item.Server` namespace but only imported `Amium.Items`.
`MqttTopicMapping.cs` uses broker path helpers from the `Amium.Item.Server` namespace but only imported `Amium.Items`.
`MqttRemoteItemRegistry.cs` uses broker value coercion helpers from the `Amium.Item.Server` namespace but did not import that namespace.
Several host files used `System.runtime.*` instead of the correct `System.Runtime.*` casing.
`Core.cs` used `result.project` instead of `result.Project`. `EnhancedSignalRuntimeManager.cs` and the legacy filtered manager used `EnhancedSignalruntime` instead of `EnhancedSignalRuntime`.
Several source files used `Hornetstudio` instead of the existing `HornetStudio` namespace and resource URI casing. Several PascalCase identifiers had the `Runtime` suffix changed to `runtime`.
`EnhancedSignalsControl.axaml.cs` used `runtime` where the existing property is `Runtime`.

## Attempted Fixes
- Added `using Amium.Item.Server;` to `MqttItemServerAdapter.cs` -> The original CS0246 errors are resolved.
- Added `using Amium.Item.Server;` to `MqttTopicMapping.cs` -> `dotnet build src/Amium.Item.Server.Mqtt/Amium.Item.Server.Mqtt.csproj --no-restore` succeeds with 0 warnings and 0 errors.
- Added `using Amium.Item.Server;` to `MqttRemoteItemRegistry.cs` -> Pending solution build verification.
- Restored `System.Runtime.*` casing in host usings -> The host project builds.
- Restored `ProjectBuildResult.Project` and `EnhancedSignalRuntime` casing -> The host project builds.
- Restored `HornetStudio`, `System.Runtime`, `ApplicationExplorerRuntime`, `EnhancedSignalRuntime`, and broker runtime variable casing across source/XAML project files -> The editor project builds.
- Restored `EnhancedSignalRow.Runtime` references -> `dotnet build HornetStudio.sln --no-restore` succeeds with 0 warnings and 0 errors.

## Current Hypothesis (optional)
- The namespace imports were missed while changing the MQTT namespace to `Amium.Item.Server.Mqtt`.
- A broad casing transformation also changed case-sensitive .NET namespaces and existing project identifiers. Those identifiers must keep their original PascalCase casing even when runtime item paths use strict snake case.

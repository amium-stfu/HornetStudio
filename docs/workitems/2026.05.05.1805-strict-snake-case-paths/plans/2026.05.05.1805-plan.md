# Strict Snake Case Paths Plan

## Goal

Introduce strict snake_case handling for all item paths and all path-relevant item property keys without compatibility aliases or migration support.

## Scope

- Convert generated item path segments to snake_case.
- Convert default item property keys to snake_case: `name`, `path`, `value`, `meta`.
- Require broker, registry, MQTT, editor target, and runtime path helpers to emit and compare canonical snake_case paths.
- Update built-in defaults, YAML definitions, tests, templates, and documentation that describe new path behavior.
- Do not migrate existing persisted layouts or support legacy key aliases such as `Properties["Value"]`.

## Architecture Direction

The path rule should be centralized and then reused by higher layers. `Amium.Item` should own strict item path and default property key behavior. Broker and UI helpers should normalize external inputs into the same canonical snake_case path form before storing, comparing, publishing, or resolving paths.

Default property keys should become canonical data keys, not only display path labels. Code must use `Properties["value"]`, `Properties["path"]`, `Properties["name"]`, and `Properties["meta"]` everywhere. PascalCase property keys should fail naturally as missing keys.

## Path Rules

- Segment separator remains dot notation for canonical in-process paths.
- Input separators `/`, `\`, and `.` should normalize to dot-separated segments.
- Each segment must be converted to snake_case before joining.
- Examples:
  - `studio.default_layout.UdlClient1.set.request` -> `studio.default_layout.udl_client1.set.request`
  - `runtime.item_broker.BrokerWidget1` -> `runtime.item_broker.broker_widget1`
  - `Status.AttachOptions` -> `status.attach_options`
  - `Value` -> `value`
  - `WritePath` -> `write_path`
- Empty path segments remain invalid for broker/item addresses.

## Property Key Rules

- Canonical default keys are `name`, `path`, `value`, and `meta`.
- Path-sensitive metadata keys should also become snake_case where they are stored as item properties, for example:
  - `Writable` -> `writable`
  - `WritePath` -> `write_path`
  - `BrokerPath` -> `broker_path`
  - `LocalPath` -> `local_path`
  - `PublishMode` -> `publish_mode`
  - `PublishIntervalMs` -> `publish_interval_ms`
- No compatibility mapping from old keys to new keys.
- Existing code references to PascalCase property keys must be updated or removed.

## Sequencing

1. Define one canonical snake_case conversion strategy for path segments and property keys.
2. Apply it first in `Amium.Item`, because all downstream behavior depends on item paths and default keys.
3. Update broker path normalization and MQTT topic mapping to use the same canonical form.
4. Update host registry and runtime path helpers.
5. Update editor target helpers and widget-generated paths.
6. Update YAML defaults, templates, docs, and tests.
7. Run focused tests first, then full available builds/tests.

## Decisions

- Strict implementation, no aliases.
- No persisted layout migration.
- Existing PascalCase path examples and property keys are considered obsolete after implementation.
- Breaking API behavior is accepted for property dictionary access and persisted path names.

## Open Questions

- Exact snake_case algorithm should be confirmed during implementation for abbreviations such as `MQTT`, `UDL`, `CSV`, `SQL`, and `ID`.
- Decide whether external protocol names in display text may remain uppercase while stored path/property segments become snake_case.

## Verification Strategy

- Add or update unit tests for:
  - item constructor path generation
  - child item path generation
  - `Repath`
  - default property keys
  - broker path normalization
  - MQTT topic mapping
  - editor target path normalization
  - host registry lookup with canonical snake_case paths
- Update hard-coded test paths to snake_case.
- Run relevant project test commands and builds after implementation.

## Risks

- This is a breaking change across public item property access and path strings.
- Case-insensitive comparisons can hide incomplete conversion if tests only check equality loosely.
- MQTT topic names will change because path segments become lowercase snake_case.
- YAML defaults and templates must be updated together with runtime code to avoid newly generated invalid paths.

# IMPLEMENTATION HANDOFF

## Goal
Complete the `ItemProperty` timestamp API rename so `LastUpdate` no longer exists and `Epoch` is the only timestamp name used by Amium.Item and HornetStudio code.

## Scope
Included:

- Remove `LastUpdate` from `Amium.Items.ItemProperty`.
- Use `Epoch` as the only public timestamp member on `ItemProperty`.
- Replace all `.LastUpdate` references in Amium.Item, Amium.Item.Mesh, Amium.Item samples, and HornetStudio code with `.Epoch`.
- Update `ItemExtension` serialization and cloning to use `Epoch`.
- Update directly related documentation that names `LastUpdate`.
- Rebuild Amium.Item artifacts before building HornetStudio.

Explicitly not included:

- Changing timestamp type or unit.
- Changing timestamp semantics.
- Adding compatibility aliases.
- Adding new dependencies.
- Release versioning or publishing.
- Broad item model changes beyond the timestamp rename.

## Starting Point
- `amium_item/src/Amium.Item/Item.cs` currently contains an intermediate state with both `LastUpdate` and `Epoch`.
- `ItemExtension.cs` still references `itemProperty.LastUpdate`.
- Several HornetStudio and Amium.Item consumers still reference `.LastUpdate`.
- HornetStudio consumes Amium.Item through artifact DLLs under `artifacts/amium_item/<configuration>/`, so stale artifacts can cause runtime mismatches.
- A related debug note exists under `docs/workitems/2026.05.08.2125-amium-item-read-invariant/debug/2026.05.08.2153-debug.md`.

## Tasks
1. Update `ItemProperty` in `Item.cs` to remove the `LastUpdate` field entirely.
2. Keep a single public `Epoch` member that stores the Unix timestamp in milliseconds.
3. Update all timestamp assignments inside `ItemProperty.Value` and the `ItemProperty` constructor to assign `Epoch`.
4. Update `ItemExtension.ItemPropertyToJsonString()` to emit `"Epoch"` instead of `"LastUpdate"`.
5. Update `ItemExtension.CloneItemProperty()` to copy `Epoch`.
6. Update all `.LastUpdate` usages under `amium_item/` and `src/` to `.Epoch`.
7. Update docs that explicitly mention `LastUpdate`.
8. Search for remaining `LastUpdate` references and remove or rename them when they refer to `ItemProperty`.
9. Build `amium_item/Amium.Item.sln --no-restore` to refresh artifacts.
10. Build `HornetStudio.sln --no-restore`.
11. Run focused tests that cover item serialization, cloning, host registry updates, and UDL/monitor flows when feasible.

## File-Level Changes
- `amium_item/src/Amium.Item/Item.cs` -> remove `LastUpdate`, keep only `Epoch`, update timestamp writes.
- `amium_item/src/Amium.Item/ItemExtension.cs` -> serialize and clone `Epoch`.
- `amium_item/src/Amium.Item.Mesh/Controllers/mesh_demo_node_controller.cs` -> replace `ItemProperty.LastUpdate` usages with `Epoch`.
- `amium_item/samples/amium_item_server_multimaster_demo/Controllers/DemoNodeController.cs` -> replace `ItemProperty.LastUpdate` usages with `Epoch`.
- `src/HornetStudio.Host/HostRegistries.cs` -> replace `ItemProperty.LastUpdate` usages with `Epoch`.
- `src/HornetStudio.Host/UiFolderContext.cs` -> replace `ItemProperty.LastUpdate` usages with `Epoch`.
- `src/HornetStudio.Host/Legacy/UiPageContext.cs` -> replace `ItemProperty.LastUpdate` usages with `Epoch`.
- `src/HornetStudio/docs/data-flow-and-signals.md` -> update `LastUpdate` wording if still relevant.

## Implementation Order
1. Change `ItemProperty` to expose only `Epoch`.
2. Update `ItemExtension` serialization and cloning.
3. Update Amium.Item.Mesh and Amium.Item sample consumers.
4. Update HornetStudio host consumers.
5. Update directly related documentation.
6. Run `rg "LastUpdate" amium_item src` and handle remaining matches carefully:
   - Rename matches that refer to `ItemProperty`.
   - Keep unrelated names such as `LastUpdatedUtc` or `LastUpdateUtc` if they are separate concepts.
7. Build Amium.Item to refresh artifacts.
8. Build HornetStudio.
9. Run focused tests where available.

## Technical Constraints
- Chat responses must be in German.
- Code, comments, XML documentation, UI text, file names, and technical identifiers must be in English.
- Newly created path-relevant names must use snake_case.
- Do not add compatibility aliases; `LastUpdate` must not remain on `ItemProperty`.
- Do not rename unrelated concepts such as `LastUpdatedUtc`, `LastUpdateUtc`, or UI formatting names unless they directly refer to `ItemProperty.LastUpdate`.
- Keep timestamp unit as Unix milliseconds.
- Do not introduce new dependencies.
- Public API change is intentional and should be mentioned as a breaking change in the final implementation summary.

## Acceptance Criteria
- `ItemProperty` no longer contains `LastUpdate`.
- `ItemProperty` exposes only `Epoch` for the update timestamp.
- No source code references `.LastUpdate` for `ItemProperty`.
- `ItemExtension` serializes `"Epoch"`.
- Cloned item properties preserve `Epoch`.
- Amium.Item builds successfully.
- HornetStudio builds successfully after Amium.Item artifacts are refreshed.
- HornetStudio no longer fails because of a missing `LastUpdate` member.

## Verification
- Run `rg "\\.LastUpdate\\b|\"LastUpdate\"" amium_item src`.
- Run `dotnet build amium_item/Amium.Item.sln --no-restore`.
- Verify `artifacts/amium_item/debug/Amium.Item.dll` is refreshed.
- Run `dotnet build HornetStudio.sln --no-restore`.
- Run relevant tests if available, especially tests touching item serialization, host registry value updates, monitor data, and UDL paths.

## Out of Scope
- Backward-compatible JSON migration for old `"LastUpdate"` payloads.
- Persisted data migration.
- The separate `read` invariant cleanup.
- MQTT topic redesign.
- Release publishing.
- NuGet packaging.

## Risks / Watchouts
- JSON payloads that still contain `"LastUpdate"` will not be automatically migrated unless explicit migration logic is added in a separate task.
- Removing `LastUpdate` is a breaking public API change for any external consumers.
- Artifact references can hide source/API mismatches. Always rebuild Amium.Item artifacts before verifying HornetStudio.
- Search results for `LastUpdatedUtc` and `LastUpdateUtc` are likely unrelated and should not be blindly renamed.

# PLAN

## Topic
BrokerWidget body layout refresh for scalable list presentation, theme-safe colors, and compact row status display.

## Problem
The current `BrokerClientControl` body uses two vertically stacked list sections with card-like multi-line rows and fixed `MaxHeight` values. This does not scale well when `Attach To UI` or `Published Items` contain many entries. In addition, row foreground and status colors are currently hard-coded in the broker row models, so theme changes can produce inconsistent or unreadable visuals.

## Goals
- Split `Attach To UI` and `Published Items` into equal-height body sections.
- Replace multi-line row cards with compact single-line list rows.
- Show status as a left-side traffic-light indicator.
- Move `Summary` into the row `ToolTip`.
- Append `Alert` to the `ToolTip` only when an alert exists.
- Keep section boundaries visually distinct while following theme rules.
- Remove static row text colors and make row visuals react correctly to theme changes.

## Scope
Included:
- `BrokerClientControl.axaml` body layout and item templates
- Broker row presentation model updates in `BrokerClientControl.axaml.cs`
- Theme-refresh wiring for broker rows

Excluded:
- Changes to broker connection behavior
- Changes to attach/publish business rules
- New dialogs or command behavior for `Edit` / `Delete`
- Broader editor theming refactors outside the broker widget

## Current State
- The body layout is defined in `src/HornetStudio.Editor/Widgets/Broker/BrokerClientControl.axaml`.
- Attached and published rows are built in `RebuildAttachedItemRows()` and `RebuildPublishedItemRows()` in `src/HornetStudio.Editor/Widgets/Broker/BrokerClientControl.axaml.cs`.
- Broker row view models currently expose hard-coded `Brushes.*` values for borders and foregrounds.
- Other widgets such as `UdlClient` already use owner-item-driven theme values and explicit `RefreshTheme()` patterns.

## Target Structure
- Keep the existing connection summary at the top of the body.
- Replace the current body grid under the summary with two section containers in a `Grid` that uses `RowDefinitions="*,*"`.
- Each section should contain:
  - a title/header
  - a themed bordered content area
  - a `ScrollViewer`
  - a compact `ItemsControl`
  - an empty-state text that remains readable and themed
- Each row should contain:
  - a small left-aligned circular status indicator
  - a single-line path label with trimming
  - `Delete` and `Edit` buttons on the right
  - a `ToolTip` composed from `SummaryText` and optional `AlertText`

## Design Decisions
1. Use section-level visual separation instead of heavy per-row cards.
2. Preserve semantic state through the traffic-light dot rather than visible multi-line secondary text.
3. Keep the row text itself theme-neutral by binding it to effective body and muted colors from the owner item.
4. Reserve semantic colors primarily for the status indicator and, if needed, subtle section accents.
5. Preserve current row data generation logic as much as possible, changing presentation rather than broker behavior.

## Row Status Model
- Attached rows:
  - Green: currently live
  - Yellow/Orange: optional future state if partial or degraded states are introduced
  - Red: persisted but not currently live
- Published rows:
  - Green: active entries exist and local root exists
  - Yellow/Orange: configured root exists but has no active entries
  - Red: selected local root is not currently available

## Sequencing
1. Redesign the body layout in AXAML for equal-height sections and compact rows.
2. Extend broker row models to expose compact text, tooltip text, status indicator properties, and theme-aware brushes.
3. Add theme refresh propagation so the rows react when the owning item theme changes.
4. Verify empty states, text trimming, and list scaling behavior.

## Risks
- If row theme refresh is missed, the body may still display stale colors after theme changes.
- Compact single-line rows increase reliance on `ToolTip`, so tooltip text composition must stay informative.
- Section height balance can regress if the inner layout still uses fixed row heights or `MaxHeight` constraints.

## Acceptance Direction
- Both list sections always share the available vertical body space 50/50.
- Rows render as one line with left traffic-light dot, center path, right action buttons.
- Tooltip shows summary and only includes alert information when present.
- Theme switching updates text, borders, and row presentation correctly.

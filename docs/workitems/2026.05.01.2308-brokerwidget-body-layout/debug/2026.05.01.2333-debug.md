# DEBUG REPORT

## Problem
The new `Attach` action in the `BrokerWidget` header opens an `Attach Items` dialog that does not show broker subitems such as `.Raw`, while the existing properties workflow does show them.

## Expected Behavior
Opening broker attach selection from the widget header should show the same live attach options and subitems as opening `AttachToUi` from the properties editor.

## Actual Behavior
The header dialog only shows persisted selected entries and marks them as missing, while the properties dialog shows the full live attach option list including subitems.

## Error Messages / Logs
No runtime exception observed. The issue is behavioral and caused by option-refresh differences.

## Relevant Code
- `src/HornetStudio.Editor/Widgets/Broker/BrokerClientControl.axaml.cs`
  - header action creates a standalone `EditorDialogField`
  - opens `AttachItemsEditorDialogWindow` with that standalone field
- `src/HornetStudio.Editor/ViewModels/MainWindowViewModel.cs`
  - `RefreshEditorDialogChoiceOptions()` continuously refreshes attach-item options for fields that belong to the main properties editor dialog
- `src/HornetStudio.Editor/ViewModels/EditorDialogField.cs`
  - `RebuildBrokerAttachEntries()` merges `Options` with selected values

## Attempted Fixes
- Reused the same dialog type and same broker option extraction logic from the existing properties workflow -> Result: structure looked correct, but live options still differed.
- Compared broker option helper methods between the property dialog path and the header path -> Result: helper logic was effectively equivalent and not the actual root cause.
- Traced attach-item entry rebuilding and option refresh flow -> Result: revealed that the properties workflow receives ongoing option refreshes, while the header-created field is only a static snapshot.

## Current Hypothesis (optional)
- Root cause: the new header action creates a detached `EditorDialogField` snapshot that is not part of the main editor dialog refresh pipeline.
- The existing properties workflow benefits from `MainWindowViewModel.RefreshEditorDialogChoiceOptions()`, which repeatedly refreshes `AttachItemList` fields from `OptionsFactory`.
- Broker attach options are published asynchronously from runtime state. When the header dialog opens, its standalone field can still hold an empty or partial option set and never gets refreshed afterward.

# IMPLEMENTATION HANDOFF

## Goal
Refactor the BrokerWidget body so `Attach To UI` and `Published Items` scale evenly in height, use compact one-line rows with a left status dot, and follow theme rules during runtime theme changes.

## Scope
Included:
- Broker body layout update
- Compact row templates for attached and published lists
- Tooltip-based summary/alert presentation
- Theme-aware broker row colors and status indicator presentation

Not included:
- New attach/publish behavior
- Changes to edit/delete actions
- Host-wide theme system changes
- Changes to unrelated widgets

## Starting Point
- Current body layout in `src/HornetStudio.Editor/Widgets/Broker/BrokerClientControl.axaml` uses one summary block plus stacked attach/publish sections in a single grid.
- Current row templates are multi-line bordered cards.
- Attached rows are built in `RebuildAttachedItemRows()`.
- Published rows are built in `RebuildPublishedItemRows()`.
- Broker row classes currently use static `Brushes.White`, `Brushes.Gray`, `Brushes.ForestGreen`, `Brushes.DimGray`, and `Brushes.LightSteelBlue`.
- Theme-safe row patterns already exist in `src/HornetStudio.Editor/Widgets/UdlClient/UdlClientControl.axaml.cs`.

## Tasks
1. Replace the broker body section layout with two equal-height themed section containers below the connection summary.
2. Convert both `ItemsControl` templates from multi-line cards to one-line list rows.
3. Add a small left circular status indicator to each broker row.
4. Move `SummaryText` into row tooltips and append `AlertText` only when it exists.
5. Replace hard-coded row foreground and border brushes with owner-item-driven theme values plus explicit semantic status colors for the dot.
6. Ensure row objects refresh their exposed theme properties when the owner item theme changes.
7. Keep empty-state text readable and still visible when a section has no items.

## File-Level Changes
- `src/HornetStudio.Editor/Widgets/Broker/BrokerClientControl.axaml` -> Redesign body layout, section containers, compact row templates, tooltip bindings, and status-dot visual.
- `src/HornetStudio.Editor/Widgets/Broker/BrokerClientControl.axaml.cs` -> Update row construction, row model properties, tooltip composition, theme-derived brush exposure, and theme refresh behavior.

## Implementation Order
1. Update the AXAML layout to remove the current stacked row/card structure and introduce two equal-height sections.
2. Add or refactor broker row model members needed by the new templates:
   - compact display text/path
   - tooltip text
   - status brush
   - theme-derived text/border/background values
3. Pass the owning `FolderItemModel` into attached and published row objects so they can use effective theme values.
4. Add `RefreshTheme()`-style methods to broker row types and invoke them when theme-relevant item properties change.
5. Adjust row rebuild methods to populate the new row model constructor arguments.
6. Run a focused build or tests relevant to the editor project.

## Technical Constraints
- Follow existing Avalonia patterns already used in the editor project.
- Keep user-visible UI text in English.
- Avoid unrelated refactors outside the broker widget files.
- Prefer existing effective theme properties such as `EffectiveBodyForeground`, `EffectiveMutedForeground`, `EffectiveBodyBorder`, and `EffectiveBodyBackground`.
- Semantic colors for the traffic-light dot may be explicit, but they must remain readable in light and dark themes.
- Do not keep static row text colors.

## Acceptance Criteria
- `Attach To UI` and `Published Items` consume equal vertical space inside the body area.
- Each list row renders on a single line with a left status dot, trimmed path text, and right action buttons.
- `SummaryText` is no longer shown inline.
- `AlertText` is not shown inline and appears only in the tooltip when present.
- Row text and borders adapt correctly when the editor theme changes.
- Empty-state text still appears for both sections when applicable.

## Verification
- Build command: run a focused editor build, for example the existing local `dotnet build` command for `src/HornetStudio.Editor/HornetStudio.Editor.csproj` or the solution if preferred.
- Manual UI verification:
  - open a BrokerWidget with several attached and published items
  - confirm both sections stay 50/50 while resizing the widget
  - confirm long paths trim instead of wrapping
  - confirm tooltip contents include summary and only include alert text when an alert exists
  - switch theme and confirm text/border colors update correctly

## Out of Scope
- Broker protocol changes
- Dialog redesigns
- Additional status categories beyond what the current data can express cleanly
- Reworking other widgets for visual consistency in the same task

## Risks / Watchouts
- Avoid leaving `MaxHeight` constraints in place, or the 50/50 split will not behave as intended.
- Do not bind the row foreground to fixed brushes; theme switching must update without rebuilding the whole widget manually.
- Keep the tooltip content concise; if summary and alert are both present, format them clearly without duplicating text.
- Preserve current sorting and item identity so edit/delete actions still target the correct rows.

- The current published-row states already distinguish active-count and root availability; use that information to map traffic-light colors without changing publish logic.
- Section-level color separation should stay subtle and theme-derived; the traffic-light dot is the primary semantic indicator.

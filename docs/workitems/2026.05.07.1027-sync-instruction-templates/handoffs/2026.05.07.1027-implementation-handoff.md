# IMPLEMENTATION HANDOFF

## Goal
Synchronize `agents.md.template` and `copilot-instructions.md.template` with the current `AGENTS.md` rules while keeping both template files solution-neutral.

## Scope
Included:

- Treat `AGENTS.md` as the source of truth.
- Update `agents.md.template` to match the current mode rules, workflow rules, workitem layout, PLAN handoff-only behavior, and DEBUG workitem/history behavior.
- Update `copilot-instructions.md.template` to reflect the current implementation-focused subset of the master rules.
- Keep templates reusable across solutions by using placeholders such as `{{SolutionName}}` instead of concrete repository names or machine paths.

Explicitly not included:

- Changing `AGENTS.md` behavior unless a contradiction is found that blocks template synchronization.
- Creating code changes.
- Creating or modifying workitems unrelated to this template sync.
- Running application projects.

## Starting Point
- `AGENTS.md` is the master and already contains the updated `PLAN` rule: create only an implementation handoff, no separate `plan.md` unless explicitly requested.
- `AGENTS.md` is the master and already contains the updated `DEBUG` rule: identify the matching workitem first, read existing debug reports, and avoid repeated disproven fixes or analysis loops.
- `agents.md.template` still contains older PLAN rules that require `plans/` and a separate plan file.
- `agents.md.template` still contains older DEBUG rules that do not require explicit workitem identification or prior debug report review.
- `copilot-instructions.md.template` is currently very short and still mentions `plans/` in workitem layout.

## Tasks
1. Compare `AGENTS.md` against `agents.md.template`.
2. Update `agents.md.template` so its structure and rules match the master where appropriate.
3. Keep `agents.md.template` solution-neutral:
   - Keep `# AGENTS.md instructions for {{SolutionName}}`.
   - Use `{{SolutionName}}.sln` where the master uses `HornetStudio.sln`.
   - Avoid machine-specific paths such as `b:\HornetStudio`.
4. Update the PLAN section in `agents.md.template`:
   - Handoff-only planning.
   - No separate `plan.md` unless explicitly requested.
   - Chat response stays short.
   - Required PLAN file is only the handoff path.
   - Recommended workitem layout contains `debug/` and `handoffs/`, not `plans/`.
5. Update the DEBUG section in `agents.md.template`:
   - Require known workitem before debugging.
   - Ask for the matching handoff or switch to `PLAN` if no workitem is known.
   - Read existing `debug/` reports before analysis.
   - Avoid repeating disproven hypotheses, failed fixes, or loops.
6. Compare `AGENTS.md` against `copilot-instructions.md.template`.
7. Update `copilot-instructions.md.template` as a concise Copilot-oriented implementation template:
   - Keep it shorter than `agents.md.template`.
   - Include German chat / English code and docs rule.
   - Include task scope, minimal changes, existing conventions, no silent architecture decisions, tests, build, documentation, secrets, and dependency/license constraints.
   - Replace old `plans/` mention with handoff-oriented workitem guidance.
   - Include DEBUG behavior only as a compact rule if Copilot is expected to work in debug context.

## File-Level Changes
- `agents.md.template` -> synchronize with current `AGENTS.md` while preserving solution-neutral placeholders.
- `copilot-instructions.md.template` -> update concise Copilot rules to align with the master without copying the full agent mode system.

## Implementation Order
1. Patch `agents.md.template` from the current `AGENTS.md` content, replacing solution-specific values with template placeholders.
2. Patch `copilot-instructions.md.template` with a compact implementation-focused rule set aligned to the master.
3. Review both files for stale references to `plans/`, `plan.md` requirements, `HornetStudio`, or `b:\HornetStudio`.
4. Run a text search to verify the old PLAN and DEBUG wording is gone.

## Technical Constraints
- Chat responses must be in German.
- Template content should remain English because it is instruction/documentation text.
- Do not introduce new dependencies.
- Do not modify project code.
- Keep path examples solution-neutral.
- Preserve existing placeholder style `{{SolutionName}}`.

## Acceptance Criteria
- `agents.md.template` reflects the current `AGENTS.md` mode behavior.
- `agents.md.template` no longer requires a separate `plans/` folder or `plan.md` for `PLAN`.
- `agents.md.template` contains the updated DEBUG workitem and debug-history rules.
- `copilot-instructions.md.template` no longer references stale `plans/` workflow.
- Both templates avoid concrete `HornetStudio` or machine-specific root paths.
- The updated templates are concise enough to be reusable across solutions.

## Verification
- Search for stale PLAN references:
  - `rg -n "Store plan files|Required PLAN Files|plans/|plan.md" agents.md.template copilot-instructions.md.template`
- Search for solution-specific values:
  - `rg -n "HornetStudio|b:\\\\HornetStudio" agents.md.template copilot-instructions.md.template`
- Optionally inspect diffs:
  - `git diff -- agents.md.template copilot-instructions.md.template`

## Out of Scope
- Applying the templates to another repository.
- Changing the current `AGENTS.md` master rules.
- Deleting previously created plan files in existing workitems.
- Updating release documentation.

## Risks / Watchouts
- `copilot-instructions.md.template` should not become a full duplicate of `agents.md.template`; it should remain focused on implementation guidance.
- `agents.md.template` should not accidentally include repository-specific paths or names from `AGENTS.md`.
- Removing all `plans/` references is intended for new PLAN workflow, but existing historical workitems may still contain `plans/`.

## Relevant Files
- `AGENTS.md`
- `agents.md.template`
- `copilot-instructions.md.template`

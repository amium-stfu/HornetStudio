# IMPLEMENTATION HANDOFF

## Goal
Flatten the MQTT client namespace and source layout by removing the nested `Amium.Item.Client.Mqtt` namespace/folder while keeping the public MQTT-specific type names.

Target public API shape:
- `Amium.Item.Client.MqttRemoteItemClient`
- `Amium.Item.Client.MqttItemClientSession`
- `Amium.Item.Client.MqttItemClientOptions`
- `Amium.Item.Client.MqttRemoteItemRegistry`

This removes the redundant full name pattern `Amium.Item.Client.Mqtt.MqttRemoteItemClient`.

## Scope
Included:
- Move MQTT client source files from `amium_item/src/Amium.Item.Client/Mqtt/` to `amium_item/src/Amium.Item.Client/`.
- Change MQTT client namespaces from `Amium.Item.Client.Mqtt` to `Amium.Item.Client`.
- Keep public class names with the `Mqtt` prefix.
- Align public type file names with public type names.
- Update all active source references from `using Amium.Item.Client.Mqtt;` to `using Amium.Item.Client;`.
- Update aliases such as `Amium.Item.Client.Mqtt.MqttItemClientOptions` to `Amium.Item.Client.MqttItemClientOptions`.
- Update current user-facing docs that describe the namespace.
- Verify builds.

Not included:
- Do not rename public classes to remove the `Mqtt` prefix.
- Do not change behavior of `MqttRemoteItemClient`, `MqttItemClientSession`, `MqttRemoteItemRegistry`, or `MqttItemClientProtocol`.
- Do not introduce transport-neutral abstractions.
- Do not split MQTT code into a new project.
- Do not rewrite historical workitem notes unless they are part of current active documentation.
- Do not change sample project names such as `Amium.Item.Client.MqttDemoClient`; those names describe the sample, not the client namespace.

## Starting Point
- Current source files are under `amium_item/src/Amium.Item.Client/Mqtt/`.
- Current namespace is `Amium.Item.Client.Mqtt`.
- Current local file names appear to be:
  - `ClientOptions.cs`
  - `ItemClientProtocol.cs`
  - `ItemClientSession.cs`
  - `ItemRegistry.cs`
  - `RemoteItemClient.cs`
- Public type names still include `Mqtt...`.
- Active source references exist in Host, Mesh, tests, samples, tools, and docs.
- `MqttItemClientProtocol` is internal shared MQTT protocol logic and should stay as a separate internal type, but it no longer needs a nested `Mqtt` namespace.

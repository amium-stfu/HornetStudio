# IMPLEMENTATION HANDOFF

## Goal
Remove the remaining in-memory state-broker leftovers from the lean MQTT server path, especially `InMemoryItemServerStore`, and finish the cleanup so `Amium.Item.Server` no longer carries a retained item-state store as part of the normal server architecture.

## Scope
Included:
- Treat `InMemoryItemServerStore` as an old state-broker implementation detail.
- Remove `InMemoryItemServerStore` if no production code requires it after the latest simplification.
- Remove or isolate `InMemoryItemServer` if it only exists to support old tests or compatibility paths.
- Remove remaining `IItemServer` dependencies from the normal `MqttItemServerHost` / server host path.
- Update tests that still instantiate `InMemoryItemServer` to the retained MQTT client-to-client workflow, or delete obsolete state-broker tests.
- Keep the lean server role: MQTT host, technical health, no item-state control.

Not included:
- Do not reintroduce server-side retained item state.
- Do not add a new state-store abstraction.
- Do not add transport abstractions.
- Do not redesign client-side item reconstruction.
- Do not change MQTT topic semantics unless required by removing dead server-state code.

## Starting Point
- `InMemoryItemServerStore` is only referenced by `InMemoryItemServer`.
- `InMemoryItemServer` is still present in `Amium.Item.Server` and still owns subscriptions, ownership tracking, retained snapshots, and update routing.
- `MqttItemServerHost` still has an optional `IItemServer? broker` constructor parameter and exposes `Broker`.
- `MqttItemServerAdapter` still stores `_broker` and exposes `StartAsync(IItemServer broker, ...)`.
- `ItemServerHealthPublisher` still has an `IItemServer`-based constructor.
- `amium_item/tests/Amium.Item.Server.Tests/Program.cs` still contains many old `InMemoryItemServer` tests.
- `IItemServer` and `ItemOwnershipConflictException` appear to have already moved to `amium_item/src/Amium.Item/Server/Contracts.cs`, so check current file locations before editing.

## Tasks
1. Run a fresh reference audit:
   - `InMemoryItemServerStore`
   - `InMemoryItemServer`
   - `RetainedItemCount`
   - `IItemServer`
   - `IItemServerStore`
   - `ItemOwnershipConflictException`
   - `MqttItemServerAdapter`
   - `ItemServerHealthPublisher`
2. Confirm whether any production code still needs `InMemoryItemServer`.
   - If only tests use it, remove the type and update/delete those tests.
   - If production code still needs it, migrate that production code first to `MqttItemServerHost` or client retained MQTT behavior.
3. Remove `InMemoryItemServerStore.cs`.
   - Remove it together with `InMemoryItemServer.cs` if the server-state broker is no longer used.
   - Do not leave an unused retained-state store behind.
4. Remove or simplify `InMemoryItemServer.cs`.
   - Preferred: delete it if no production path needs it.
   - If kept temporarily, mark it clearly as legacy/test-only and remove it from default server APIs.
5. Finish server host cleanup:
   - Remove `IItemServer? broker` from `MqttItemServerHost` if no longer needed.
   - Remove `Broker` property if it only exposes old state-broker internals.
   - Ensure host startup does not require an item broker.
6. Finish adapter cleanup:
   - Remove `MqttItemServerAdapter` if it only bridges MQTT to old `IItemServer` state.
   - If a tiny adapter remains for direct retained health publishing or test publishing, rename/reshape it so it no longer implements state-broker concepts.
7. Clean health publishing:
   - Remove `IItemServer` constructor from `ItemServerHealthPublisher` if no current call path requires it.
   - Keep only direct retained MQTT health publishing.
   - Remove retained item count if it depends on `InMemoryItemServer`.
8. Update tests:
   - Delete old state-broker tests for retained server store, ownership, and server subscription fanout.
   - Keep or add tests for:
     - `MqttItemServerHost` starts/stops.
     - retained health topics are published.
     - client A publishes retained item data.
     - client B reconstructs retained item state.
   - Update `HornetStudio` host/editor tests that still create old server adapter/broker instances.
9. Update project and docs:
   - Remove deleted files from project if SDK globbing does not handle it automatically.
   - Update architecture/usage docs if they still mention `InMemoryItemServer`.
   - Update `CHANGELOG.md` if this cleanup is tracked as a breaking internal/API simplification.
10. Build and run relevant tests.

## Technical Constraints
- Chat responses must be German; code, comments, XML documentation, UI text, file names, and identifiers must be English.
- Do not add new dependencies.
- Do not add abstractions for hypothetical future needs.
- Keep server hot path simple.
- Preserve client-side retained MQTT behavior.
- Keep changes scoped to server-state cleanup and related tests/docs.

## Acceptance Criteria
- `InMemoryItemServerStore.cs` is removed, or there is a documented concrete production reason it remains.
- The default `Amium.Item.Server` path does not instantiate or expose `InMemoryItemServer`.
- `MqttItemServerHost` starts without an `IItemServer`.
- `ItemServerHealthPublisher` does not require an `IItemServer`.
- Old server retained-state tests are removed or replaced by MQTT retained client reconstruction tests.
- Reference audit shows no unintended production dependency on `InMemoryItemServerStore`.
- Builds succeed.

## Risks / Watchouts

- If `IItemServer` is still part of public client abstractions, remove it only after migrating those abstractions.
- Health item count may disappear or need a new cheap metric; do not keep `InMemoryItemServerStore` only for this metric.
- `MqttItemServerAdapter` may still be used as a test publisher; replace with a smaller test helper if needed.

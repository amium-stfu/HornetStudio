  - Result: stopped the mirror update storm and allowed cross-write/runtime propagation to converge.
- Fixed observed MQTT import so value and property topics update an existing item when possible instead of replacing the retained item with a partial snapshot.
  - Result: prevents `(null)` value regressions when observed inbound import is enabled.

## Current Hypothesis (optional)
- Confirmed root cause:
  - Mirror candidates were deduplicated by signature and source timestamp. The remote registry refreshes `LastUpdate` when it rebuilds incoming MQTT topics, so identical mirrored values appeared newer on every loop. That caused repeated retained mirror publishes and delayed or overwrote live cross-write/runtime updates.
  - The later full-baseline regression came from importing every observed MQTT property topic as a full snapshot. Non-value property snapshots contained the default `read = null`, replacing previously observed values with `(null)`.
- Current status:
  - The mesh self-test is green with 36/36 passing cases.
  - Suspicious update growth is gone in the fixed run.

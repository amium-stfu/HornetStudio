# PLAN

## Goal

Add a dedicated executable host project that starts the default MQTT Item Server by calling `ItemServerServiceHost.RunAsync`, while keeping `Amium.Item.Server` as a reusable library.

## Planning Validation

No major planning blockers remain.

The TODO defines the missing executable, the intended host entry point, the library boundary, and the related files. Configuration extensibility is intentionally out of scope for this first host executable.

## Scope

- Create one small console project for the Item Server host executable.
- Reference `Amium.Item.Server` from the host project.
- Add a minimal async `Program.cs` that forwards process lifetime to `ItemServerServiceHost.RunAsync`.
- Add the new project to `HornetStudio.sln`.
- Verify with a full solution build.

## Architecture

The existing `Amium.Item.Server` project should remain the runtime/library owner. The new executable should only be a thin process entry point and should not duplicate MQTT startup logic, server options, broker behavior, or client-facing APIs.

Use the folder name `src/amium_item_server_host/` to comply with the current snake_case path rule for newly created path-relevant names. Keep .NET identities in the established style:

- Project name: `Amium.Item.Server.Host`
- Assembly name: `Amium.Item.Server.Host`
- Root namespace: `Amium.Item.Server.Host`

This keeps file-system naming compliant without forcing non-standard C# identifiers.

## Plan

1. Create `src/amium_item_server_host/Amium.Item.Server.Host.csproj` as a `net9.0` executable project.
2. Configure nullable reference types and implicit usings consistently with nearby Amium projects.
3. Add a project reference to `../Amium.Item.Server/Amium.Item.Server.csproj`.
4. Create `src/amium_item_server_host/Program.cs` with an async entry point that calls `ItemServerServiceHost.RunAsync`.
5. Use `Console.CancelKeyPress` or equivalent cancellation handling only if the existing host entry point does not already receive process cancellation cleanly from the default console lifetime.
6. Add the host project to `HornetStudio.sln` and nest it under the existing Amium solution folder if practical.
7. Run `dotnet build HornetStudio.sln --no-restore`.
8. Update the TODO status only if the repository has an established pattern for completed TODO handling; otherwise leave it for explicit cleanup.

## Major Decisions

- Do not move `ItemServerServiceHost` out of `Amium.Item.Server`.
- Do not add configuration files, command-line parsing, hosting framework dependencies, logging infrastructure, or publish settings in this task.
- Do not add tests unless host startup logic grows beyond a simple pass-through entry point.

## Acceptance Criteria

- The solution contains a startable `Amium.Item.Server.Host` executable project.
- Running the host starts the default MQTT item server through the existing `ItemServerServiceHost.RunAsync`.
- `Amium.Item.Server` remains a library project.
- Client projects do not gain server host behavior.
- The full solution builds without new errors.

# IMPLEMENTATION HANDOFF

## Goal

Add a dedicated executable project that starts the default MQTT Item Server host by calling `ItemServerServiceHost.RunAsync`.

## Scope

Included:

- Create a small .NET console executable project for `Amium.Item.Server.Host`.
- Reference the existing `Amium.Item.Server` library project.
- Add an async process entry point that starts the existing service host.
- Add the new project to `HornetStudio.sln`.
- Verify with a full solution build.

Explicitly not included:

- Moving server runtime code out of `Amium.Item.Server`.
- Adding configuration, command-line parsing, logging infrastructure, service installation, publish profiles, or new runtime dependencies.
- Reintroducing server behavior into client packages.

## Starting Point

- `src/Amium.Item.Server/ItemServerServiceHost.cs` already exposes `public static Task RunAsync(CancellationToken cancellationToken = default)`.
- `src/Amium.Item.Server/Amium.Item.Server.csproj` is a `net9.0` library project and should stay a library.
- No dedicated executable currently starts the default MQTT item server host.
- The repository rule for new path-relevant names requires snake_case paths.

## Tasks

1. Create a new directory `src/amium_item_server_host/`.
2. Create `src/amium_item_server_host/Amium.Item.Server.Host.csproj`.
3. Set project properties:
   - `OutputType` = `Exe`
   - `TargetFramework` = `net9.0`
   - `Nullable` = `enable`
   - `ImplicitUsings` = `enable`
   - `AssemblyName` = `Amium.Item.Server.Host`
   - `RootNamespace` = `Amium.Item.Server.Host`
4. Add a `ProjectReference` to `..\Amium.Item.Server\Amium.Item.Server.csproj`.
5. Create `src/amium_item_server_host/Program.cs`.
6. Implement a minimal async console entry point that calls `ItemServerServiceHost.RunAsync`.
7. Add cancellation handling for Ctrl+C only if needed to pass a cancellation token into `RunAsync`; keep it local and simple.
8. Add the new project to `HornetStudio.sln`.
9. Prefer nesting the project under the existing `Amium` solution folder if the solution structure supports it cleanly.
10. Run the full build verification.

## File-Level Changes

- `src/amium_item_server_host/Amium.Item.Server.Host.csproj` -> define the new executable project and reference `Amium.Item.Server`.
- `src/amium_item_server_host/Program.cs` -> start `ItemServerServiceHost.RunAsync`.
- `HornetStudio.sln` -> include the new project, preferably under the existing Amium solution folder.

## Implementation Order

1. Add the new host project file.
2. Add the minimal program entry point.
3. Add the project to the solution.
4. Build the solution.
5. Adjust only if build errors reveal project metadata or namespace issues.

## Technical Constraints

- Use existing .NET project style and nearby Amium project conventions.
- Do not introduce new dependencies.
- Keep code, comments, XML documentation, UI text, file names, and technical identifiers in English.
- Chat responses must remain German.
- Preserve `Amium.Item.Server` as a library.
- Do not modify client packages.
- Keep the implementation minimal and directly scoped to the executable host.

## Acceptance Criteria

- `Amium.Item.Server.Host` exists as a buildable executable project.
- The host project references `Amium.Item.Server`.
- The host starts the existing default MQTT item server via `ItemServerServiceHost.RunAsync`.
- `Amium.Item.Server` remains a library project.
- `HornetStudio.sln` includes the new project.
- The solution builds without new errors.

## Verification

- Run `dotnet build HornetStudio.sln --no-restore`.
- Do not run the host process as part of build verification unless explicitly requested.

## Out of Scope

- Runtime configuration options.
- Publish/release packaging.
- Windows service integration.
- MQTT behavior changes.
- Client package changes.
- Stress tests or integration tests.

## Risks / Watchouts

- A minimal top-level statement can be correct, but cancellation behavior may be poor if Ctrl+C does not pass cancellation into `RunAsync`; verify whether simple local `CancellationTokenSource` handling is needed.
- Adding the project to the solution manually can require a new GUID and solution folder nesting metadata.
- The new path must use snake_case, but C# project identity should remain `Amium.Item.Server.Host`.

# DEBUG REPORT

## Problem
The request value still falls back to the previous value when a SignalWidget is attached. More runtime visibility is needed to identify which component writes the old value.

## Expected Behavior
An incoming MQTT value update should flow into the HostRegistry once and the SignalWidget should refresh from that value without publishing an older local value back.

## Actual Behavior
The exact writer of the old value is still unknown. The behavior looks like a timing or feedback issue where a SignalWidget or publisher path writes after the MQTT write-back.

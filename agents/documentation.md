# Documentation Rules

## Code Documentation

- Document newly added or behaviorally changed public methods with XML summaries and parameters.
- Create or maintain XML documentation for newly added or behaviorally changed public classes.

## Markdown Documentation

- Update relevant Markdown documentation when setup, behavior, structure, or usage changes.
- Maintain `AGENTS.md` with repository agent rules.
- Update `CHANGELOG.md` for releases, user-visible behavior changes, public API changes, or documented workflows.
- Maintain `TODO.md` or `ROADMAP.md` only when already present or clearly useful.
- Use `docs/todos/` for standalone backlog entries created through `TODO` mode.
